# WIN-1706 Option 1 Workstream

This document turns `WIN-1706` Option 1 into an executable workstream:

- **promotion means library/runtime alignment**
- Windows is only first-class supported when the **library** and the **CLI** tell the same story
- `Sandbox::support_info()` and `Sandbox::apply()` must match the supported Windows runtime behavior

Use this as the handoff artifact for the next PR candidate series.

## Goal

Promote Windows from:

- supported CLI surface + partial library contract

to:

- one aligned Windows support contract across:
  - `crates/nono`
  - `crates/nono-cli`
  - tests
  - CI
  - docs and release messaging

## Non-Goals

This workstream does **not** require:

- implementing live `nono shell` on Windows
- implementing live `nono wrap` on Windows
- pretending Windows has full Seatbelt/Landlock feature parity
- broadening support claims beyond the Windows surface already enforced honestly by the runtime

The goal is alignment, not overclaiming.

## Hard Definition of Done

`WIN-1706` Option 1 is complete only when:

1. `Sandbox::support_info()` no longer reports Windows as `partial` for the supported Windows contract.
2. `Sandbox::apply()` on Windows no longer returns the generic `UnsupportedPlatform` path for the contract the CLI already supports.
3. CLI support output no longer needs a “CLI supported, library partial” split.
4. Windows docs and README can describe Windows as first-class supported without qualification about a narrower embedded library boundary.
5. Verification passes:
   - `cargo fmt --all -- --check`
   - `cargo build --workspace`
   - `cargo clippy --workspace --all-targets --all-features -- -D warnings -D clippy::unwrap_used`
   - `cargo test --workspace`
   - Windows regression harness
   - docs/release checks

## Dark Factory Rules

If another LLM executes this workstream, require these constraints:

- Work in PR-sized slices only; do not attempt the entire Option 1 conversion in one branch without checkpoints.
- Preserve the repo’s security mandates:
  - no silent fallback
  - fail closed on unsupported Windows shapes
  - no broadening path or network access to satisfy tests
- Prefer restoring branch-compatible Windows behavior when upstream auto-merges Unix-only assumptions over Windows files.
- Treat Windows host failures separately from source failures:
  - `LNK1318`
  - `os error 112`
  - package cache lock contention
- Do not update public support language before the library contract is honest.
- Every PR candidate must include explicit review notes answering the blocking review questions above.
