<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resolved engine exe + interpreter paths → launch policy | An uncovered interpreter spawned by a covered wrapper = partial confinement (the core ENG-02 threat) |
| workspace directory NTFS security descriptor → mandatory-label relabel | A non-relabelable (admin-owned) workspace makes confined writes fail opaquely (R-B3) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-05 | Elevation of Privilege / Tampering | covered wrapper spawns UNCOVERED interpreter (partial confinement) | mitigate | Extend the single coverage gate to coverage-check every declared/resolved interpreter; fail-secure NonoError naming the exact interpreter + wrapper + `--allow` fix; no auto-mutation. Acceptance: `validate_launch_paths_refuses_uncovered_interpreter` green |
| T-71-06 | Tampering / EoP | admin-owned workspace → relabel fails opaquely → operator disables confinement to "make it work" | mitigate | `path_has_write_owner` lets Plan 04 refuse BEFORE spawn with a named diagnostic (the correct WRITE_OWNER predictor, stronger than owner-SID equality); no auto-`takeown`. Acceptance: helper returns Ok(false) on non-WRITE_OWNER paths, true for user dirs |
| T-71-07 | Tampering | string `starts_with` path comparison (`/home` matches `/homeevil`) | mitigate | Reuse `covers_path` (component-wise `windows_paths_start_with_case_insensitive`, mod.rs:522) for every interpreter check — NEVER string ops (CLAUDE.md footgun #1) |
| T-71-08 | Tampering | symlink/junction in workspace path changes between check and relabel (TOCTOU) | accept | Canonicalize at the boundary (normalize_candidate_path); `try_set_mandatory_label` re-verifies WRITE_OWNER at apply time — the pre-gate is an early named refusal, the apply-time re-check is the enforcement boundary. Residual TOCTOU window accepted (low value, requires local write to the workspace path mid-launch) |
| T-71-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; library-only edits. No `[ASSUMED]`/`[SUS]` packages |
</threat_model>
