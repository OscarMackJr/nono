<objective>
Wire daemon-side security telemetry (DRAIN-04). Make the `SecurityEventLayer` reachable
from the standalone `nono-agentd` binary, register it once at startup in both service and
foreground modes, thread the already-read `policy.telemetry` config through the SOLE HKLM
read, and add a non-host-gated test proving an in-process `nono_security::network_deny`
event advances the HMAC chain.

Purpose: Today `nono-agentd` sets up NO tracing subscriber, so daemon-launched agent
denials emit nothing. This closes the daemon-path telemetry gap with real wired code.
Output: A registered daemon `SecurityEventLayer`, a minimal daemon-side init helper, an
extended SOLE-read return surfacing `TelemetryConfig`, a `chain_sequence()` accessor, and a
passing integration-style test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HKLM machine policy → daemon | Admin-controlled `policy.telemetry` (`enabled`, `min_severity`) crosses into the daemon process; must be honored, not bypassed |
| in-process proxy → SecurityEventLayer | `nono_security::network_deny` events cross from the proxy into the registered telemetry layer |
| SecurityEventLayer → OS sink (Windows) | Scrubbed/hashed security events cross into ETW + Application Log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-90-01 | Tampering / Repudiation | Daemon telemetry event chain | mitigate | Per-session HMAC-SHA256 chain reused verbatim from SecurityEventLayer; D-01 test asserts the chain head advances per event (sequence == 1) |
| T-90-02 | Information disclosure | Event Log payload | mitigate | Reuse existing path-hashing + scrub_value in on_event; proxy emit deliberately omits `reason`; no new fields added |
| T-90-03 | Elevation / policy bypass | Admin telemetry opt-out | mitigate | Daemon honors `config.enabled` opt-out + `min_severity` threshold sourced from the SOLE HKLM read; opt-out test asserts a disabled layer does NOT advance the chain |
| T-90-04 | Denial of service / fail-open | Policy load error | mitigate | resolve_machine_egress_policy Err → abort (fail-secure) preserved unchanged; absent → TelemetryConfig::default() (default-ON), never silently off |
| T-90-05 | Tampering | Second HKLM read (TOCTOU / drift) | mitigate | No new read_machine_egress_policy call; telemetry config threaded from the existing SOLE read (Phase 83 D-04 contract) |
| T-90-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase; all types reuse already-pinned workspace crates (RESEARCH.md: N/A Package Legitimacy Audit) |
</threat_model>
