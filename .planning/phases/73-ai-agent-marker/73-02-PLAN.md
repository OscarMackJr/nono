```rust
// SDDL → SD → OwnedSecurityDescriptor guard pattern:
let sddl = format!("S:(ML;;0x{mask:X};;;LW)");
let wide_sddl: Vec<u16> = sddl.encode_utf16().chain(std::iter::once(0)).collect();
let mut security_descriptor: PSECURITY_DESCRIPTOR = std::ptr::null_mut();
let ok = unsafe {
    ConvertStringSecurityDescriptorToSecurityDescriptorW(
        wide_sddl.as_ptr(), SDDL_REVISION_1, &mut security_descriptor, std::ptr::null_mut(),
    )
};
if ok == 0 { return Err(NonoError::SandboxInit(...)); }
let _sd_guard = OwnedSecurityDescriptor(security_descriptor);  // LocalFree on Drop
// Then use security_descriptor as lpSecurityDescriptor in SECURITY_ATTRIBUTES
```

SDDL for job ACL (D-03 intent, hex rights per RESEARCH.md Pitfall 3 — use hex, not SDDL mnemonics):
  Base: "D:P(A;;0x1F001F;;;OW)(D;;0x1F001F;;;LW)"
  With package_sid: append "(D;;0x1F001F;;;<package_sid_sddl>)"
  D:P = protected DACL (no inheritance)
  (A;;0x1F001F;;;OW) = ALLOW 0x1F001F (JOB_OBJECT_ALL_ACCESS) to Object Owner
  (D;;0x1F001F;;;LW) = DENY 0x1F001F to Low Integrity label
  (D;;0x1F001F;;;<package_sid>) = DENY 0x1F001F to the agent's own package SID

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| launch path → CreateJobObjectW | Job object SD is built from trusted SDDL before the untrusted child exists; no attacker input enters the SDDL at this stage |
| confined child → job object | Child token is Low-IL AppContainer; SD explicitly denies Low-IL any job access (the boundary this plan enforces) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-07 | Tampering | Agent calling TerminateJobObject on its own job | mitigate | Explicit DACL denies Low-IL label (LW) and optionally the agent's own package SID all JOB_OBJECT_ALL_ACCESS (0x1F001F). MIC Low-IL → Medium-IL kernel check is the first gate; explicit DACL is belt-and-suspenders |
| T-73-08 | Tampering | Breakaway from job to shed confinement | mitigate | JOB_OBJECT_LIMIT_BREAKAWAY_OK is NOT set (already true pre-Phase-73; negative test job_never_has_breakaway_ok makes this an enforced regression guard) |
| T-73-09 | Elevation of Privilege | Agent opening job by name to gain capabilities | accept | Opening the job handle by name does NOT confer job membership or identity — membership is established at AssignProcessToJobObject spawn time; the job ACL blocks job-object modification, not membership queries |
| T-73-10 | Spoofing | SDDL injection via package_sid string | mitigate | package_sid_str is produced by ConvertSidToStringSidW (kernel-validated SDDL SID string, e.g., "S-1-15-2-...") — it is not attacker-controlled. The SDDL string is built programmatically, not from user input |
| T-73-SC | Tampering | npm/pip/cargo installs | accept | No new external package dependencies — uses only already-pinned windows-sys 0.59 |
</threat_model>
