        1. OpenProcess(PROCESS_QUERY_LIMITED_INFORMATION, 0, pid) — on null HANDLE return Err(NonoError::SandboxInit(...)) with GLE
        2. Wrap in OwnedHandle from crate::sandbox::windows::OwnedHandle (closes on drop)
        3. OpenProcessToken(h_process.raw(), TOKEN_QUERY, &mut h_token) — on 0 return return Err(NonoError::SandboxInit(...))

        6. Allocate Vec<u8> of size needed as usize
        7. Second GetTokenInformation call with buf.as_mut_ptr() — on 0 return return Err(NonoError::SandboxInit(...))
        8. Cast buf.as_ptr() as *const TOKEN_APPCONTAINER_INFORMATION, read info.TokenAppContainer
        9. If info.TokenAppContainer.is_null(): return Ok(None)
        10. Extract string WHILE BUF IS ALIVE: call ConvertSidToStringSidW on info.TokenAppContainer to get the SDDL string.
            Use the same ConvertSidToStringSidW pattern as package_sid_to_string in sandbox/windows.rs (lines 789-820).

        11. return Ok(Some(sid_str))

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| classify(pid) caller → Win32 kernel | PID is a u32 passed by an operator/daemon; kernel owns the actual process token |
| AgentRegistry (private in-memory set) → classification result consumer | Registry is authoritative; classification output is not independently verifiable by the consumer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-01 | Spoofing | `read_process_appcontainer_sid(pid)` | mitigate | Opens real kernel process token for the supplied PID via `OpenProcess` — cannot be lied to about PID-to-token mapping; kernel enforces the binding |
| T-73-02 | Spoofing | SID namespace pattern (`nono.session.<guess>`) | mitigate | 122-bit random UUID suffix makes the name unguessable; AUTHZ check is `minted_sids.contains(sid_str)` — namespace pattern match is NEVER used for authz (D-02) |
| T-73-03 | Spoofing | Self-made AppContainer with correct-namespace name | mitigate | Registry check rejects any SID not in `minted_sids`, regardless of namespace. A self-made AppContainer (even named `nono.session.<valid-uuid>`) is rejected because its SID is absent from the registry — only the launcher's private mint path inserts |
| T-73-04 | Elevation of Privilege | Opening named job by name to claim `AI_AGENT` identity | accept (not applicable here) | Job membership is established only by `AssignProcessToJobObject` at spawn time — opening the job handle by name does NOT confer membership or identity. Authz is the token SID, not job membership (D-02) |
| T-73-05 | Denial of Service | Mutex poisoning on `AgentRegistry` | mitigate | PoisonError on `Mutex::lock()` propagates as `Err(NonoError::SandboxInit(...))` via `.map_err` — fail-secure, never panics |
| T-73-06 | Information Disclosure | PSID double-free / use-after-free in `read_process_appcontainer_sid` | mitigate | Buffer-internal PSID is NEVER wrapped in `OwnedAppContainerSid` (which calls FreeSid on drop). String form extracted while `Vec<u8>` buffer is alive; buffer dropped after string is cloned. `#[allow(clippy::unwrap_used)]` is test-only — never in production classify path |
| T-73-SC | Tampering | npm/pip/cargo installs | accept | No new external package dependencies in this plan — all Win32 APIs accessed via already-pinned windows-sys 0.59 |
</threat_model>
