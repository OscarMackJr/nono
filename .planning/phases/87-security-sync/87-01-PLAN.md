<objective>
Port upstream commit `e2086877` (fix(linux): trap sendto/sendmsg to prevent AF_UNIX datagram
bypass, v0.64.0) onto the fork's divergent Linux enforcement code. This closes a SOCK_DGRAM
bypass where `sendto`/`sendmsg`/`sendmmsg` fell through the seccomp BPF filter to
`SECCOMP_RET_ALLOW` because the filter only trapped `connect`/`bind`.

Purpose: SEC-01 — the AF_UNIX datagram bypass is structurally impossible after this plan.
A sandboxed child cannot send datagrams to arbitrary Unix-socket paths without a grant and
supervisor validation (D-01 hybrid gate: no grant = static EPERM; grant present = USER_NOTIF
route to existing supervisor validation loop).

Output: Extended BPF filters in `linux.rs`; `UnixSocketOp::Send` + send-family constants in
`capability.rs`; multi-sockaddr dispatch loop in `supervisor_linux.rs`; updated filter tests.
All as one atomic cherry-pick commit per D-04.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sandboxed child → kernel | Syscalls from the untrusted child process cross into the kernel BPF filter |
| Child address space → supervisor | Supervisor reads child memory via /proc/PID/mem to extract sockaddr/msghdr data |
| Seccomp USER_NOTIF → supervisor decision | Notification carries raw args (pointer values) that the supervisor must validate before trusting |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87-01 | Tampering | AF_UNIX SOCK_DGRAM bypass via sendto/sendmsg/sendmmsg (pre-fix) | mitigate | Expand BPF filters to trap all three syscalls; route to USER_NOTIF for grant-present validation; static EPERM for no-grant path (D-01). Closed by this plan. |
| T-87-02 | Elevation of Privilege | Abstract-namespace AF_UNIX sockets bypassing pathname allowlist | mitigate | classify_af_unix() returns UnixSocketKind::Abstract → denied by decide_network_notification for all send-family ops (existing path, confirmed still applied after port) |
| T-87-03 | Denial of Service | sendmmsg with large vlen (up to 2^32-1) causing unbounded /proc/PID/mem reads | mitigate | MAX_MMSGHDRS = 1024 cap in read_mmsghdr_dests; overflow-safe arithmetic with checked_mul/checked_add |
| T-87-04 | Information Disclosure | TOCTOU: child changes sockaddr between BPF trap and supervisor read | mitigate | notif_id_valid check before decision; if pid changes or notif expires → deny. Existing mitigation; confirmed applied after multi-sockaddr refactor. |
| T-87-05 | Tampering | Null/zero msg_name in sendmsg/sendto (connected socket) being misclassified as a bypass | accept | Explicit fast-path: if args[4]==0 (sendto) or msg_name==NULL (sendmsg/sendmmsg) → continue_notif. Connected-socket sends carry no per-call destination; no bypass possible. |
