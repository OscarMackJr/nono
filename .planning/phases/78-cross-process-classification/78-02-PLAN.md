## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nono CLI process → control pipe | nono classify sends a classify frame; the pipe SDDL enforces Medium-IL minimum (SC3, inherited from Plan 01 daemon side). |
| Classify response → caller output | The JSON/human output function must not include SID data regardless of the daemon's internal registry state (SC4). |
| Structural fallback path | When the daemon is absent, run_classify uses the Phase 73 non-authoritative path — this is deliberate and labelled; it must not be confused with the authoritative path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-06 | Information Disclosure | classify_daemon_request response printing | mitigate | Output function omits SID fields; only verdict enum string is printed. SC4 acceptance criterion asserts no "package_sid" in output. |
| T-78-07 | Spoofing | Fallback mis-labelled as authoritative | mitigate | Fallback (run_classify) retains its existing `authoritative: false` JSON flag and NOTE_* disclaimers unchanged (no modification to classify_runtime.rs). Only the daemon-sourced response carries `authoritative: true`. |
| T-78-08 | Elevation of Privilege | Low-IL caller on client side | accept | Low-IL caller cannot open the control pipe (SDDL, T-78-04). Even if a Low-IL process runs nono classify, the CreateFileW call fails at the kernel before the classify payload is sent. |
| T-78-09 | Denial of Service | Pipe timeout on classify request | accept | 5-second timeout already set in windows_control_pipe_request (TIMEOUT_MS=5_000). On timeout, falls through to structural fallback — non-fatal. |
| T-78-SC | Tampering | npm/pip/cargo installs | accept | No new package installs. |
