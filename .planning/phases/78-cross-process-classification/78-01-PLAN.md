<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → control pipe | Any process that can open `\\.\pipe\nono-agentd-control` sends JSON frames. The Medium-IL SACL (`ML;;NW;;;ME`) means only Medium+ IL callers (interactive user, non-AppContainer) can reach this boundary at all. |
| handle_classify → agent_registry | The handler reads `DaemonState::agent_registry` (daemon's authoritative in-memory SID set). The registry is controlled exclusively by the daemon's launch/reap path — no caller can inject a SID. |
| response → caller | The response is a plain string verdict. SC4 threat: the response must not contain SID data that a caller (who did not launch the queried agent) should not see. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-01 | Spoofing | ControlRequest deserialization | mitigate | serde `#[serde(tag="action")]` rejects unknown action values; `from_slice` error path returns framed error and closes connection (existing control_loop.rs pattern). |
| T-78-02 | Tampering | agent_registry during classify | mitigate | acquire agent_registry lock before reading; Mutex ensures exclusive access during the classify call. Lock order: agent_registry before tenants (mod.rs:262) respected. |
| T-78-03 | Information Disclosure | Classify response leaks package_sid (SC4) | mitigate | handle_classify returns only "AiAgent" or "NotAnAgent" — no SID string. The AiAgent arm destructures with `_` (discarded). Test `classify_response_contains_no_sid_field` asserts the invariant. |
| T-78-04 | Elevation of Privilege | Low-IL caller reaching control pipe (SC3) | mitigate | Structural: `CONTROL_PIPE_SDDL` SACL `S:(ML;;NW;;;ME)` bars Low-IL opens at the kernel — no code change needed, no new SDDL. Test `control_pipe_sddl_is_medium_il_only` asserts the constant is unchanged. |
| T-78-05 | Denial of Service | Oversized classify request frame | mitigate | `MAX_CONTROL_FRAME` bound already enforced in `handle_control_connection` before dispatch — applies to classify requests identically to existing verbs (T-74-07-04). |
| T-78-SC | Tampering | npm/pip/cargo installs | accept | No new package installs; composition over existing crate graph. |
</threat_model>
