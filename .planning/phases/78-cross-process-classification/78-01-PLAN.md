<objective>
Add the `Classify` verb to the daemon's existing control pipe: a new `ControlRequest::Classify { pid: u32 }` variant, a pure `classify_response_string` function that maps `AgentClassification` to the verdict string, a `handle_classify` function that reads the daemon's live `agent_registry`, and the dispatch arm wiring it into `handle_control_connection`. Expose `handle_classify_testable` and add four unit tests that serve as the unattended `cargo test --bin nono-agentd -- classify` gate.

The key structural requirement (BLOCKER 1): `AgentRegistry::classify(pid)` calls `read_process_appcontainer_sid(pid)` first — an OS call — so any test running against a non-AppContainer process will always return `NotAnAgent` regardless of registry state. To make the AiAgent path and the SC4 no-SID assertion deterministically testable in a unit test (no real AppContainer process needed), factor the response construction into a PURE function `classify_response_string(classification: &AgentClassification) -> String` that takes the already-resolved classification and maps it to the verdict string. Tests call `classify_response_string` directly, bypassing the OS call. The `handle_classify_testable` shim tests only the registry-miss path honestly (a non-AppContainer test PID always returns NotAnAgent).

Purpose: CLAS-01 and CLAS-02 both require the daemon to answer authoritative classification queries. SC3 (Low-IL caller denied) is free because the existing `CONTROL_PIPE_SDDL` Medium-IL SACL (`ML;;NW;;;ME`) already blocks Low-IL openers at the kernel. SC4 (no cross-tenant SID disclosure) is enforced by the verdict-only response shape: no package_sid field is emitted, ever.

Output: Modified `control_loop.rs` with the Classify verb, pure response fn, handler, testable shim, and 4 unit tests. The file is the sole artifact — no new files.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → control pipe | Any process that can open `\\.\pipe\nono-agentd-control` sends JSON frames. The Medium-IL SACL (`ML;;NW;;;ME`) means only Medium+ IL callers (interactive user, non-AppContainer) can reach this boundary at all. |
| handle_classify → agent_registry | The handler reads `DaemonState::agent_registry` (daemon's authoritative in-memory SID set). The registry is controlled exclusively by the daemon's launch/reap path — no caller can inject a SID. |
| response → caller | The response is a plain string verdict. SC4 threat: the response must not contain SID data that a caller (who did not launch the queried agent) should not see. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-01 | Spoofing | ControlRequest deserialization | mitigate | serde `#[serde(tag="action")]` rejects unknown action values; `from_slice` error path returns framed error and closes connection (existing control_loop.rs pattern). |
| T-78-02 | Tampering | agent_registry during classify | mitigate | acquire agent_registry lock before reading; Mutex ensures exclusive access during the classify call. Lock order: agent_registry before tenants (mod.rs:262) respected. |
| T-78-03 | Information Disclosure | Classify response leaks package_sid (SC4) | mitigate | `classify_response_string` returns only "AiAgent" or "NotAnAgent" — the AiAgent arm destructures with `_ ` (discarded). | T-78-04 | Elevation of Privilege | Low-IL caller reaching control pipe (SC3) | mitigate | Structural: `CONTROL_PIPE_SDDL` SACL `S:(ML;;NW;;;ME)` bars Low-IL opens at the kernel — no code change needed, no new SDDL. Test `control_pipe_sddl_is_medium_il_only` asserts the constant is unchanged. |

| T-78-SC | Tampering | npm/pip/cargo installs | accept | No new package installs; composition over existing crate graph. |
