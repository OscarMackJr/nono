<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dry-run script → crates.io / PyPI / npm | A mis-typed command (publish without --dry-run, twine upload) would push a real, immutable artifact during what is supposed to be validation. |
| host env → registry credentials | A token referenced in the dry-run path is both unnecessary and a leak/accidental-publish risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-97-08 | Tampering | accidental live publish during dry-run | mitigate | Only --dry-run / twine check / npm publish --dry-run used; grep gate asserts no `twine upload` and no token-bearing publish path in the script. |
| T-97-09 | Information disclosure | registry tokens in logs | mitigate | No --token / no PyPI/npm credential referenced anywhere; grep for token returns 0. |
| T-97-10 | Repudiation | silent skip of a binding leg | mitigate | Absent toolchain is a documented SKIP with verbatim signature, not a silent pass; crates leg is always-runnable and must pass. |
| T-97-03-SC | Tampering | npm/pip/cargo installs | accept | Dry-runs build from the existing lockfile / sdist; no new package is installed. maturin/npm/twine are host preconditions, not installed by a task. |
</threat_model>
