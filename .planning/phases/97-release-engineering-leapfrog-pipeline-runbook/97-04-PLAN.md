Gate contract (copy the two-function shape from scripts/gates/harness-self-check.ps1):
  Test-Precondition  -> $null (run) | "reason string" (SKIP_HOST_UNAVAILABLE)
  Invoke-Gate        -> [ordered]@{ gate; verdict; reason; detail; timestamp } with verdict PASS | FAIL
  RULES: the gate RETURNS its verdict object; it MUST NOT call exit (the runner owns exit mapping).
         A failing assertion may throw (surfaces as harness-internal error). NEVER return PASS on a
         broken precondition.
  Discovery: drop the file at scripts/gates/release-readiness.ps1 — verify-dark.ps1 auto-discovers it.
  Invocation (NOTE: -Gate, single dash; NOT --gate):
    pwsh -File scripts/verify-dark.ps1 -Gate release-readiness   (exit 0 = PASS, 2 = FAIL, 3 = SKIP, 4 = harness error)

Release-readiness assertions the gate must make (all must hold for PASS):
  - cargo metadata reports nono, nono-cli, nono-proxy, nono-shell-broker, nono-fltmgr-client, nono-ffi all at 0.66.0
  - no occurrence of 0.62.2 in any tracked workspace Cargo.toml
  - leapfrog target 0.66.0 is strictly greater than upstream highest 0.65.1
  - git staging area contains NO path under build_notes/ or .gsd/  (git diff --cached --name-only + git ls-files)
  - Cargo.lock contains the 0.66.0 workspace entries

PUBLIC-repo pre-push checklist (MUST be embedded verbatim in the runbook):
  - Repo is PUBLIC pending Microsoft minifilter-altitude approval — verify NO build_notes/ or .gsd/ staged before any push.
  - Confirm crate leapfrog >= 0.66.0 (clears upstream 0.65.1).
  - Operator push (git push of tags + cargo/twine/npm publish) is the SOLE remaining manual action.

Operator one-step push sequence (document, do NOT execute — PREPARE-ONLY):
  1. Run the dry-run orchestrator: pwsh -File scripts/release-dry-run.ps1   (must be green)
  2. Run the readiness gate: pwsh -File scripts/verify-dark.ps1 -Gate release-readiness   (must PASS)
  3. git push origin <branch> && git push origin v0.66.0   (tag push triggers release.yml)
  4. crates.io publish order: nono -> nono-proxy -> nono-shell-broker -> nono-cli  (release.yml publish-crates does this on tag; manual fallback documented)
  5. PyPI: maturin publish (from ../nono-py) ; npm: npm publish (from ../nono-ts)
