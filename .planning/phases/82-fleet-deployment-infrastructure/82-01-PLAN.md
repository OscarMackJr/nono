<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MSI installer (SYSTEM) → host machine state | A silent `/qn` install runs as SYSTEM/SCCM and writes machine-wide PATH, a trust-root cert, a service, and a policy key — all high-value, all under the most privileged token. |
| Deferred cert CustomAction (LocalSystem) → certificate trust stores | A SYSTEM-context process invokes `nono.exe setup --trust-root`, which (via `cert_trust::import_machine_root`) mutates the machine trust anchors (`Root`+`TrustedPublisher`). |
| ADMX template → HKLM\SOFTWARE\Policies\nono | The template is the operator-facing path that writes the security-policy spine. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-01 | Tampering | cert-import CustomAction `ExeCommand` | mitigate | Invoke `nono.exe` and the staged DER `.cer` RELATIVELY under `INSTALLFOLDER` (Program Files, Admins-only writable) exactly like the existing `nono.exe setup --uninstall-wfp` CA — never a user-writable or attacker-controllable path; both the exe and the cert ship inside the MSI cab, not fetched at runtime. |
| T-82-02 | Spoofing | wrong/forged cert imported into LocalMachine\Root | mitigate | The imported cert is the pinned POC root staged into the MSI harvest at build time (`<File>` component, byte-identical to the signing cert); no runtime selection. The store list (`Root`+`TrustedPublisher`) is a single Rust source of truth in `cert_trust::import_machine_root`. SUMMARY records the cert thumbprint for audit. |
| T-82-03 | Elevation of Privilege | machine-wide PATH entry points at a writable dir → PATH hijack | mitigate | The PATH `Environment` value is `[INSTALLFOLDER]` = Program Files\nono (Admins-only writable), `Part="last"` so it cannot shadow system tools; unchanged from the shipped element — assert it remains `System="yes"` + `[INSTALLFOLDER]`. |
| T-82-04 | Denial of Service | cert/service failure rolls back the whole product (1603) | mitigate | Cert CA `Return="ignore"` and `ServiceInstall Vital="no"` make both non-fatal; static-CRT removes the `0xC0000135` DLL-load rollback (DEPLOY-01); `nono health` (Plan 03) reports the degraded state instead of failing the install. |
| T-82-05 | Information Disclosure | sentinel key / ProgramData ACLs over-permissive | accept | Phase 82 sentinel holds only `InstalledByMsi=1` (no secrets); ProgramData root holds machine-global non-sensitive data (incl. the public-cert PEM bundle).
