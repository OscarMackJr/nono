<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MSI installer (SYSTEM) → host machine state | A silent `/qn` install runs as SYSTEM/SCCM and writes machine-wide PATH, a trust-root cert, a service, and a policy key — all high-value, all under the most privileged token. |
| Deferred cert CustomAction (LocalSystem) → certificate trust stores | A SYSTEM-context process invokes `certutil -addstore Root/TrustedPublisher`, mutating the machine trust anchors. |
| ADMX template → HKLM\SOFTWARE\Policies\nono | The template is the operator-facing path that writes the security-policy spine. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-01 | Tampering | cert-import CustomAction `ExeCommand` | mitigate | Reference the staged `.cer` RELATIVELY under `INSTALLFOLDER` (Program Files, Admins-only writable) exactly like the existing `nono.exe setup` CA — never a user-writable or attacker-controllable path; the cert ships inside the MSI cab, not fetched at runtime. |
| T-82-02 | Spoofing | wrong/forged cert imported into LocalMachine\Root | mitigate | The imported cert is the pinned POC root staged into the MSI harvest at build time (`<File>` component, byte-identical to the signing cert); no runtime selection. SUMMARY records the cert thumbprint for audit. |
| T-82-03 | Elevation of Privilege | machine-wide PATH entry points at a writable dir → PATH hijack | mitigate | The PATH `Environment` value is `[INSTALLFOLDER]` = Program Files\nono (Admins-only writable), `Part="last"` so it cannot shadow system tools; unchanged from the shipped element — assert it remains `System="yes"` + `[INSTALLFOLDER]`. |
| T-82-04 | Denial of Service | cert/service failure rolls back the whole product (1603) | mitigate | Cert CA `Return="ignore"` and `ServiceInstall Vital="no"` make both non-fatal; static-CRT removes the `0xC0000135` DLL-load rollback (DEPLOY-01); `nono health` (Plan 03) reports the degraded state instead of failing the install. |
| T-82-05 | Information Disclosure | sentinel key / ProgramData ACLs over-permissive | accept | Phase 82 sentinel holds only `InstalledByMsi=1` (no secrets); ProgramData root holds machine-global non-sensitive data. Egress policy values land in Phase 83 under HKLM\SOFTWARE\Policies (admin-only by GP convention). Low value this phase. |
| T-82-06 | Tampering | operator pushes one flat allow-list mixing exact + wildcard hosts (wildcard creep) | mitigate | ADMX defines SEPARATE `AllowedSuffixes` (list) and `AllowedHosts` (exact) policies (Pitfall 3); template comment documents 64-bit-hive deployment (Pitfall 7). |
| T-82-SC | Tampering | npm/pip/cargo installs | mitigate | This plan adds NO package-manager installs (PowerShell + WiX + ADMX XML only). No legitimacy gate required; slopcheck N/A. |
</threat_model>
