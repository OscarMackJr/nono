From windows-confinement-model.md:
  Sound Shape B: confine() must be called at process startup BEFORE any privileged handle is opened.
  Post-hoc IL-drop is leaky (handles opened before the drop keep their access). D-03 rejects this.
  The sound Windows equivalent: born-confined self-re-exec via nono.exe broker before main() does anything.

From engine-agnostic-confinement.md:
  Broker arm requires windows_low_il_broker:true + dev-layout or signed nono.exe (R-B4 trust gate).
  Workspace must be user-owned (R-B3). For file-only proof, network.block:false is fine.
  nono.exe run --profile langchain-python --allow <ws> -- python.exe -c "..." is the Shape A invocation.

From 72-CONTEXT.md D-03:
  Shape B: confine() bootstraps a confined re-exec of the agent through nono.exe as the FIRST thing
  main() does, BEFORE any privileged handle is opened. This is the sound Windows equivalent.

Soundness invariant to prove: When confine(profile, allow) is called:
  1. nono.exe is invoked via std::process::Command (or Python subprocess.run) BEFORE any file handle,
     socket, registry key, or other privileged object is opened in the calling python process.
  2. The resulting child process runs under a Low-IL/AppContainer token.
  3. The child can write to the allowed workspace and is denied outside it.
