<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PythonREPLTool payload → confined process | LangChain's PythonREPLTool exec() runs arbitrary Python code inside the confined process; the boundary is OS-enforced (Low-IL/AppContainer), not code-level |
| Test fixtures → nono.exe | Tests invoke nono.exe via confined_run; nono.exe applies its own coverage gate and profile validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-72-04-01 | Tampering | Deny test using a writable outside path | mitigate | Outside path uses tmp_path.parent (not a hardcoded C:\ path); the parent dir is outside the allowed workspace tree; denial is OS-enforced, not code-checked |
| T-72-04-02 | Tampering | PythonREPLTool CLM-safe constraint | accept | PythonREPLTool sends arbitrary Python string to exec(); in the confined process the CLM constraint applies only on the real Claude-Code hook path (from windows-confinement-model.md); for a standalone confined python process launched by nono.exe, CLM is not enforced; OS IL-enforcement is what matters here |
| T-72-04-03 | Denial of Service | confine() infinite re-exec loop | mitigate | NONO_ALREADY_CONFINED guard tested explicitly in test_confine_already_confined_guard; confine() returns Ok(()) if guard is set, preventing infinite re-exec |
| T-72-04-04 | Spoofing | Test with hardcoded path outside workspace | mitigate | Deny path is tmp_path.parent / "nono_outside_probe.txt" — derived from pytest's tmp_path, not hardcoded; avoids permission inconsistencies on different machines |
| T-72-04-SC | Tampering | langchain / langchain_experimental pip install | mitigate | langchain is an optional extra; example degrades gracefully without it; if pip install is done by the operator, they verify via npmjs.com/pypi equivalents; no blocking checkpoint needed (operator-installed test dependency) |
</threat_model>
