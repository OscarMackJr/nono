<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python caller → confined_run / confine | Python code supplies exe, args, allow paths, profile name as strings — all untrusted before validation |
| confined_run / confine → nono.exe | Functions invoke nono.exe via Command; nono.exe applies its own coverage gate and profile validation |
| confine() re-exec boundary | confine() re-execs the current process through nono.exe broker; NONO_ALREADY_CONFINED prevents infinite loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-72-02-01 | Tampering | confine() self-re-exec ordering (born-confined invariant) | mitigate | NONO_ALREADY_CONFINED guard checked FIRST in confine() before any other operation; documented in binding docs as "must be the first call in main()" |
| T-72-02-02 | Elevation of Privilege | confined_run allow path injection | mitigate | Each allow path string is passed verbatim to nono.exe --allow; nono.exe canonicalizes and validates at grant time (nono's own path security, not re-validated in binding layer) |
| T-72-02-03 | Spoofing | NONO_EXE env var pointing to a malicious binary | mitigate | find_nono_exe() validates the path is a file before returning it; trust is rooted in R-B4 broker-trust gate enforced by nono.exe's own signing check at runtime. Binding does not bypass the gate. |
| T-72-02-04 | Tampering | confine() NONO_ALREADY_CONFINED env bypass | mitigate | Guard uses std::env::var("NONO_ALREADY_CONFINED") == Ok("1"); string equality, not prefix match. Cannot be bypassed by setting "NONO_ALREADY_CONFINED=1extra" (guard checks exact "1"). |
| T-72-02-05 | Denial of Service | GIL held during blocking child wait | mitigate | py.detach(|| ...) releases the GIL before waiting on child completion; other Python threads are not blocked. |
| T-72-02-SC | Tampering | pip/cargo installs (cross-target drift) | mitigate | No new crate deps in windows_confined_run.rs (std::process::Command only); slopcheck + blocking human checkpoint would apply if any [ASSUMED] packages added |
</threat_model>
