<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scripts/gates/*.ps1 (dot-sourced) | Gate files are version-controlled PowerShell, dot-sourced by the runner. A rogue/malformed gate file executes in the runner's process space. |
| .nono-runtime/verdicts/ (write target) | JSON verdict files written to a repo-local directory. The aggregate path `_aggregate.json` is constructed programmatically. |
| stdout (CI consumer) | The final aggregate JSON line is the machine-readable close signal. A garbage overall verdict on stdout must never be silently consumed as PASS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-01 | Tampering | Gate file injection via rogue scripts/gates/*.ps1 | mitigate | Gate discovery uses version-controlled `scripts/gates/` directory (no user-supplied path); auto-discovery pattern `*.ps1` is fixed by the runner; a gate that calls `exit` directly is caught by the runner's dot-source contract (gate MUST NOT call exit — enforced by code review and harness-self-check pattern). |
| T-81-02 | Spoofing | Rogue gate returns unknown/garbage verdict to force silent PASS | mitigate | `Resolve-VerdictClass` (already shipping) validates verdict against known set {PASS, FAIL, SKIP_HOST_UNAVAILABLE}; any unknown value -> HARNESS_ERROR (exit 4), never PASS (CR-02). |
| T-81-03 | Tampering | Aggregate artifact path-injection via gate name | mitigate | The aggregate gate name is the string literal `'_aggregate'` hardcoded in the runner — no gate-supplied input affects this path. Per-gate paths use `$gateName` which is derived from `[System.IO.Path]::GetFileNameWithoutExtension` (no directory traversal characters). |
| T-81-04 | Elevation of Privilege | Stub *.json in .nono-runtime/verdicts/ inflating PASS count | mitigate | Gate set derived exclusively from `$discoveredGates` (built from `scripts/gates/*.ps1`); the verdicts directory is never globbed; stub files (_skip-probe.json, test-skip-stub.json, skip-verify-stub.json) cannot enter the rollup. |
| T-81-05 | Denial of Service | A gate's Invoke-Gate hangs indefinitely, blocking the all-run sweep | accept | Gates are expected to be time-bounded by their own logic (established harness contract); adding a per-gate timeout would require significant restructuring. Risk is low (all four current gates have bounded execution). |
| T-81-SC | Tampering | npm/pip/cargo installs | accept | This phase adds no new package manager installs; PowerShell-only change. No slopcheck required. |
</threat_model>
