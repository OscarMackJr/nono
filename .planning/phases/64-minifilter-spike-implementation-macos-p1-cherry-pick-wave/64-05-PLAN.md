    # nono drivers

    ## Overview
    Brief paragraph: out-of-workspace WDK MSBuild project for the Gap 6b minifilter feasibility spike.
    C/WDK driver (NOT Rust; see DESIGN.md for the windows-drivers-rs rationale). The spike .sys is
    VM-local throwaway — never committed, never MSI-bundled. The existing `nono-wfp-driver.sys`
    placeholder in `crates/nono-cli/data/windows/` is UNTOUCHED by this spike and must stay untouched.

    ## Prerequisites
    - Azure VM: Windows 11, Standard security type (NOT Trusted Launch), Secure Boot OFF, HVCI OFF
      (see Phase 63 63-SC1-vm-state.md for provisioning). TESTSIGNING ON required.
    - Toolchain on the VM: EWDK 26H1 ISO (VS Build Tools 18.3.0) OR WDK 28000.1761 + VS 2026.
      msbuild, signtool, makecert, certmgr, inf2cat must all be on PATH inside the EWDK build env.
    - Altitude: enumerate `fltmc filters` before install; pick a non-colliding number in
      FSFilter Activity Monitor band (360000–389999). Avoid AV range 320000–329998.
    - See `nono-fltmgr/DESIGN.md` for BSOD-avoidance contract and altitude details.

    ## Build the Driver

    ### 1. Clone / copy to VM
    Copy the `drivers/nono-fltmgr/` directory to the VM (e.g., `C:\nono-fltmgr`).
    Note: this directory is intentionally out-of-Cargo-workspace (WDK/MSBuild only).

    ### 2. Open EWDK build environment
    Mount the EWDK ISO and run `LaunchBuildEnv.cmd`.
    (Alternately: VS 2026 Developer Command Prompt if WDK 28000.1761 is installed.)

    ### 3. Build
    cd C:\nono-fltmgr
    msbuild nono-fltmgr.vcxproj /p:Configuration=Release /p:Platform=x64
    Expected: "Build succeeded. 0 Warning(s) 0 Error(s)"
    Output: x64\Release\nono-fltmgr.sys (do NOT commit this file)

    ### 4. Test-sign the driver

    (All commands in an elevated prompt inside the EWDK build env:)

    # Create a test code-signing cert and install it as a trusted root
    makecert -r -pe -ss PrivateCertStore -n "CN=NonoTestSign" NonoTestSign.cer
    certmgr /add NonoTestSign.cer /s /r localMachine root

    # Generate the catalog file
    inf2cat /driver:. /os:10_x64 /uselocaltime

    # Sign the catalog
    signtool sign /fd sha256 /s PrivateCertStore /n "CN=NonoTestSign" /t http://timestamp.digicert.com nono-fltmgr.cat

    # Enable test-signing mode (if not already set; requires reboot to take effect)
    bcdedit /set testsigning on
    # After reboot, re-enter the EWDK build env and verify: bcdedit /enum | findstr testsigning

    # Take a pre-load snapshot (BSOD rollback safeguard — DESIGN.md recommends this)
    # az snapshot create --resource-group <rg> --name nono-fltmgr-snap-preload --source <disk>

    ### 5. Install the driver
    pnputil /add-driver nono-fltmgr.inf /install
    Expected: "Driver package added successfully."

    ### 6. Verify the driver is loaded
    fltmc instances
    fltmc filters
    Expected: nono-fltmgr appears at the chosen altitude.

    ## Run the Rust User-Mode Policy Client (nono-fltmgr-client)

    The `crates/nono-fltmgr-client` Cargo workspace member is the user-mode policy client that
    receives IPC messages from the kernel driver over `\NonoPolicyPort`.

    ### Build (on Windows dev host or VM):
    cargo build -p nono-fltmgr-client --target x86_64-pc-windows-msvc
    Output: target/x86_64-pc-windows-msvc/debug/nono_fltmgr_client.exe

    ### Run on the VM:
    Copy `nono_fltmgr_client.exe` to the VM and run it with the deny-target path as its first
    argument. The binary runs until Ctrl-C or the port disconnects. Start it BEFORE triggering
    any CreateFile attempt on the deny-target path, then in a second terminal run the deny harness:

    nono_fltmgr_client.exe C:\nono-deny-test\secret.txt

    The policy client must be running BEFORE any CreateFile attempt on the deny-target path.
    (The driver returns STATUS_TIMEOUT / fail-open if no client is connected.)

    - `crates/nono-cli/data/windows/nono-wfp-driver.sys`: This is the WFP driver placeholder.
      It is UNTOUCHED by Phase 64. Do not confuse it with nono-fltmgr.sys.
    - The spike `nono-fltmgr.sys` binary must NOT be committed to the repository (see DESIGN.md §Security).
    - Official Microsoft altitude assignment for production use: pending fsfcomm@microsoft.com reply
      (~30 business days from 2026-06-07 request). The test-signed spike uses a temporary non-colliding
      number in the Activity Monitor band (see 64-SC1-driver-evidence.md for the chosen value).
