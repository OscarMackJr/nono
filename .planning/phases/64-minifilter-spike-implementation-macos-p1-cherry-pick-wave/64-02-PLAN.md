<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kernel pre-create callback → user-mode client | FltSendMessage carries NonoIpcRequest over \NonoPolicyPort; finite timeout prevents kernel from blocking on missing user-mode client |
| User-mode policy reply → kernel decision | NonoIpcReply.Decision (0/1) interpreted by worker thread; out-of-range values default to allow (fail-open) |
| Ring buffer (kernel) | Single-slot with KSPIN_LOCK; back-pressure: if occupied, fail-open and permit the I/O |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-01 | Denial of Service | NonoPreCreate callback (own-I/O recursion) | mitigate | No ZwCreateFile/NtCreateFile/ZwWriteFile anywhere in the driver; data copied to ring buffer and sent via FltSendMessage to user mode only — verified by `grep -c "ZwCreateFile" == 0` gate |
| T-63-02 | Denial of Service | FltSendMessage (blocking system hang) | mitigate | `timeout.QuadPart = -5000000LL` (500ms finite relative timeout); STATUS_TIMEOUT path completes IRP with STATUS_SUCCESS (fail-open for spike) per DESIGN.md T-63-02 |
| T-63-03 | Denial of Service | IRQL violation in callback (BSOD) | mitigate | `NT_ASSERT(KeGetCurrentIrql() <= APC_LEVEL)` at top of NonoPreCreate; all allocations use `ExAllocatePool2(POOL_FLAG_NON_PAGED_NX)` (never PagedPool in callback context); spinlock released BEFORE FltSendMessage |
| T-63-04 | Tampering | NonoIpcRequest kernel buffer overflow | mitigate | Fixed-size WCHAR PathBuffer[260] (MAX_PATH); `FltGetFileNameInformation` uses normalized name; path copy bounded to 259 WCHARs with null terminator; static assert `sizeof == 532` prevents silent layout drift |
| T-63-05 | Tampering | Spike .sys committed to repo | mitigate | .sys binary is VM-local throwaway; .gitignore rule ensures it cannot be accidentally staged; DESIGN.md T-63-05 documents the prohibition |
| T-64-SC | Tampering | npm/pip/cargo installs | accept | No Cargo changes in this plan; C driver build uses WDK toolchain on isolated VM only |
</threat_model>
