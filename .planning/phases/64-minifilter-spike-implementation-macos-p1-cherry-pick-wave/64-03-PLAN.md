    Implement `pub fn run_policy_client(deny_path: &str) -> Result<(), Box<dyn std::error::Error>>`:
    1. Encode port name as a null-terminated wide string:
       `let port_name: Vec<u16>` = encode "\NonoPolicyPort" as UTF-16 + '\0' terminator.
       (Use OsStr::new("\\NonoPolicyPort").encode_wide().chain(std::iter::once(0)).collect())
    2. Call FilterConnectCommunicationPort with the port name, 0 flags, null context/size/null.
       // SAFETY: port_name is a Vec<u16> null-terminated; no other thread modifies it.
       If the returned handle == INVALID_HANDLE_VALUE: use GetLastError() in the error string.
       Return Err("FilterConnectCommunicationPort failed: GLE={}".into()) with the error code.
    3. Loop:
       a. Initialize a zeroed NonoIpcRequest buffer (std::mem::zeroed() inside unsafe, or
          MaybeUninit; must include header as first field).
       b. Call FilterGetMessage(port, &mut buf.header as *mut _, size_of::<NonoIpcRequest>() as u32,
          null_mut()).
          // SAFETY: buf is a valid aligned buffer of sizeof(NonoIpcRequest) bytes.
          If return != 0 (HRESULT error): break out of loop (port disconnected or error).
       c. Decode path: `String::from_utf16_lossy(&buf.path_buffer)`, trim trailing nulls.
       d. Determine decision: if path.eq_ignore_ascii_case(deny_path) { 1u32 } else { 0u32 }.
          (deny_path comparison is case-insensitive for Windows path semantics)
       e. Build reply struct:
          `#[repr(C)] struct NonoReply { header: FILTER_REPLY_HEADER, decision: u32 }`
          Set header.Status = 0; header.MessageId = buf.header.MessageId (echo back the ID).
       f. Call FilterReplyMessage(port, &mut reply.header as *mut _, size_of::<NonoReply>() as u32).
          // SAFETY: reply is a valid aligned buffer.
          Ignore the return value for the spike (non-zero = port closed, loop will exit next iter).
    4. After loop: call CloseHandle(port). // SAFETY: port is a valid open handle.
       Return Ok(()).

    Error handling: use ? for Result propagation, if/return for FFI non-zero HRESULT, Box<dyn Error>
    for the return type. No .unwrap() or .expect() in production code.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-02 | Denial of Service | FltSendMessage timeout (user-mode client must exist for non-timeout path) | accept | Spike: fail-open on STATUS_TIMEOUT means driver continues to allow I/O if client not running; production ADR will address fail-closed policy |
