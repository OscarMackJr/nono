<objective>
Close the CPLT-01 coverage gap surfaced by the Phase 77 host proof (77-03-SUMMARY, recorded
FAIL): the runtime ancestor-RA guard walks only the confined target binary's resolution chain
(`config.resolved_program`), NOT the `--workspace` (child CWD) chain. Node engines such as the
WinGet `@github/copilot` CLI **self-extract their Node package under the workspace**
(`<workspace>\.nono-runtime\…\AC\copilot\pkg\…`), so `realpathSync`/`lstat` walks the WORKSPACE's
ancestor chain and denies (EPERM) on the user-profile dir `C:\Users\<user>` — which gets only the
TRAVERSE grant (0x21) from `AppliedAncestorTraverseGuard`, not FILE_READ_ATTRIBUTES (0x80).

Fix: extend `AppliedAncestorReadAttributesGuard` to grant RA on the user-owned ancestors of BOTH
walk targets (the binary chain AND the workspace chain), deduplicated, preserving the D-04 split
(stop at the first non-owned ancestor on each chain; C:\Users / C:\ stay covered by the CPLT-02
admin grant). Then re-run the `copilot-e2e` gate on a provisioned host for the real PASS that fully
closes CPLT-03.

Purpose: this is the runtime-guard completion that makes the confined Node/Copilot launch resolve
its modules end-to-end under AppContainer without a STATUS_ACCESS_DENIED / module-resolution crash.
Output: the extended guard + its wiring, with tests, and the host-proof PASS.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined Node engine → host filesystem ancestor chain | The RA grant widens the per-run package SID's access on user-owned ancestor DIRECTORIES. It must remain attribute-read only (0x80) and must never extend to system roots (C:\Users, C:\) at runtime. |
| ancestor-RA guard lifetime → host DACL state | Grants are per-run and MUST be reverted on Drop; a leaked grant leaves a residual ACE on the user's profile-chain dirs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-04a | Elevation of Privilege (over-grant) | multi-target RA walk | mitigate | Grant is FILE_READ_ATTRIBUTES only (0x80, via the shipped grant_sid_read_attributes_on_path), NO_INHERITANCE, on user-owned ancestor directories only; per-chain stop at the first non-owned ancestor preserves the D-04 split so C:\Users / C:\ are never touched at runtime. |
| T-77-04b | Tampering / resource leak (residual ACE) | dedup + revert | mitigate | Dedup ensures each ancestor is recorded once and reverted once (LIFO) on Drop; fail-closed revert-all + Err on any grant/ownership error; existing revoke is mask-agnostic trustee-match removal. |
| T-77-04c | Repudiation (false-PASS at the gate) | copilot-e2e verdict | accept (handled in 77-03) | The gate already orders confinement-FAIL checks before the auth-SKIP heuristic and fails-closed on nono launch errors / non-zero exit (commit 78f1101f); this plan does not weaken those checks. |
</threat_model>
