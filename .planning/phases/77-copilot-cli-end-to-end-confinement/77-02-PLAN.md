<objective>
Deliver the CPLT-02 one-time-admin setup step: a generic, idempotent, non-destructive
`nono setup --grant-ancestors --profile <p>` command that grants the well-known
`ALL APPLICATION PACKAGES` SID (`S-1-15-2-1`) `FILE_READ_ATTRIBUTES` on the system ancestors
`C:\` and `C:\Users` — the two ancestors nono structurally cannot ACL at runtime as a
non-elevated user (the runtime guard from CPLT-01 stops at them by the D-04 ownership gate).

Purpose: This closes the runtime/admin split. Every per-run AppContainer token is a member of
`ALL APPLICATION PACKAGES`, so this single durable grant covers any future run of any confined
Node engine (D-05 RESOLUTION + D-06 engine-agnostic intent) without re-granting per-run SIDs.
Output: A generic setup subcommand extension and its admin-gated, idempotent runtime.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| elevated operator → DACL of system roots (C:\, C:\Users) | A one-time-admin command edits the DACL of the most sensitive paths; over-grant, wrong principal, or destructive edit here is the highest-severity surface in the phase. |
| any confined AppContainer token → C:\ / C:\Users RA | The well-known group SID grant is visible to EVERY AppContainer on the host, not just nono's — the grant must be minimal and attribute-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-02 | Elevation of Privilege / Information Disclosure | grant scope (mask + principal + paths) | mitigate | Mask is `FILE_READ_ATTRIBUTES` only (from CPLT-01 primitive — attribute-read, not content); principal is the well-known `ALL APPLICATION PACKAGES` group SID (never the interactive user / Everyone); paths are exactly `C:\` and `C:\Users` — asserted by acceptance criteria (D-09). |
| T-77-02b | Tampering (DACL bloat / destructive edit) | re-run idempotency + non-destructiveness | mitigate | GetAce/EqualSid absence check before grant (no duplicate ACE); never remove/modify an existing ACE; never touch a deny-ACE — both asserted by `grant_ancestors_idempotent` and `grant_ancestors_non_destructive` unit tests. |
| T-77-02c | Spoofing (wrong durable grantee) | grantee SID derivation | mitigate | Hardcoded well-known `S-1-15-2-1`; the runtime is forbidden from deriving the grantee from the per-run package SID (D-05 RESOLUTION) — asserted by the no-derive acceptance criterion. |
| T-77-02d | (Defense-in-depth) fail-open on insufficient privilege | admin gate | mitigate | `is_admin_process()` gate returns `NonoError::Setup` elevation error; `SetNamedSecurityInfoW` access-denied surfaces a clear "requires elevation" error (fail-closed, never silent partial grant). |
| T-77-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan. |
</threat_model>
