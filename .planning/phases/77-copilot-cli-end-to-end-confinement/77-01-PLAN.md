<objective>
Deliver the CPLT-01 runtime fix: nono auto-grants `FILE_READ_ATTRIBUTES` (RA) on every
user-ownable ancestor of the confined target's resolution chain at launch time, so the
Node-ESM `realpathSync`/`lstat` ancestor walk that the standalone `@github/copilot` CLI
performs succeeds under AppContainer instead of being denied with `STATUS_ACCESS_DENIED`.
Also wire `node.exe` interpreter coverage into the `copilot-cli` profile (D-01/D-02) so the
Node Copilot CLI passes the executable-coverage gate at all.

Purpose: This is the structural fix that turns the v2.12 Phase 75 "confine-only" Copilot
re-scope into real end-to-end confinement. It is the foundation CPLT-02 (admin grant) and
CPLT-03 (gate) build on.
Output: A new RA DACL primitive in the library, a new ownership-gated ancestor-RA RAII guard
wired into the Windows launch path, and an updated `copilot-cli` profile with corrected tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined AppContainer child → host filesystem ancestors | The per-run package SID is granted RA on ancestors so Node-ESM lstat succeeds; over-grant or wrong-principal here is a confinement/visibility risk. |
| nono-cli launch path → DACL of user-owned dirs | The runtime guard edits DACLs of ancestors it owns; a fail-open here would leave stray grants. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-01 | Elevation of Privilege / Information Disclosure | `grant_sid_read_attributes_on_path` mask | mitigate | Mask is exactly `FILE_READ_ATTRIBUTES` (0x80) — attribute-read only, NOT `FILE_GENERIC_READ`; asserted by the const value and the no-broader-bits acceptance criterion (D-09). |
| T-77-01b | Tampering (confinement bypass) | `AppliedAncestorReadAttributesGuard` ownership gate | mitigate | `path_is_owned_by_current_user` gate (path-component-correct, never `starts_with`); `Ok(false) => break` so the runtime guard provably never edits `C:\Users`/`C:\` as a non-elevated user (D-04 structural split). |
| T-77-01c | (Defense-in-depth) fail-open on grant error | guard apply/revert | mitigate | `Err =>` revert_all + propagate (never swallow); `edit_dacl_for_sid` fails closed at every step; LIFO best-effort revert on Drop. |
| T-77-01d | Spoofing / coverage bypass | copilot-cli interpreter coverage | accept | The executable-coverage gate (R-B3) fail-secure refuses to launch an uncovered interpreter; adding `node.exe` is the required minimal coverage, not a widening — uncovered shim paths surface as a clear coverage error (settled empirically in CPLT-03, not here). |
| T-77-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan — no external dependency surface added. |
</threat_model>
