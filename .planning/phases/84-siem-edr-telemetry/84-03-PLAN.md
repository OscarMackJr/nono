    Create docs/adr/telemetry-tamper-evidence.md as a concise ADR with these mandatory sections:

    Title: "ADR: Telemetry Tamper-Evidence Scope (Phase 84)"

    Status: Accepted

    Context: nono Phase 84 ships HMAC-SHA256 chained security events to the Windows Application Event Log. The in-session HMAC provides ordering/continuity proof within a single nono session. This ADR records the exact scope of this guarantee and what it does NOT cover.

    Decision:
    1. Tamper boundary = Windows Event Forwarding. Once events are forwarded by WEF/SIEM agent to an external SIEM (Splunk, Sentinel), the copy is outside a locally-compromised host's reach. This is the tamper-evidence claim.
    2. In-session HMAC chain only. The HMAC key is a per-session OsRng-generated ephemeral key held in memory (Zeroizing<[u8;32]>), never persisted to disk, registry, or Event Log. The chain proves intra-session event ordering. On session end, the key is zeroized; no cross-session continuity.
    3. What is NOT guaranteed: (a) A local admin can clear the Application Event Log (`wevtutil cl Application`) without a forensic trace — this removes events before WEF has forwarded them. (b) A local admin with HKLM write could inject a fake nono.exe that produces events with a fresh HMAC key — the SIEM would see a new key and a gap but cannot distinguish injection from a normal session restart. (c) The chain does not survive nono process restart.

    Consequences:
    - Fleet deployments SHOULD configure WEF subscriptions to forward the Application log to a SIEM collector to achieve effective tamper-evidence.
    - For cryptographic-local tamper anchoring (TPM PCR-sealed key or remote key management), see SEED-005 (future milestone). This is the correct scope for that work.
    - Do NOT claim "cryptographically tamper-proof audit log" in documentation. Correct claim: "structured, HMAC-sequenced security events, tamper-evident via Windows Event Forwarding."

    The ADR does NOT need to be long — accuracy matters more than length. Record the decision faithfully per D-07.
