<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registry → TelemetryConfig | HKLM telemetry sub-section values are admin-controlled but could be malformed or wrong type |
| Path → PathHash | Raw canonical paths enter the hash function; must never flow to the event payload unencoded |
| Free-text fields → SecurityEvent | Reason/label strings from denial sources may contain tokens or URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-01 | Information Disclosure | TelemetryConfig parse — registry REG_SZ values | mitigate | D-14: malformed REG value → stderr eprintln + TelemetryConfig::default(); never propagate raw registry bytes into SecurityEvent |
| T-84-02 | Information Disclosure | path_hash_for — raw canonical path input | mitigate | D-08: SHA-256(session_salt ‖ canonical_path) truncated to 16 bytes hex; raw path never in any SecurityEvent field; SC-3 gate greps for raw path strings |
| T-84-03 | Information Disclosure | free-text fields (reason, label, hook_name) in SecurityEvent | mitigate | D-10: nono::scrub_value applied to all free-text fields before SecurityEvent construction; host stays cleartext per SC-1 only |
| T-84-04 | Tampering | HMAC key in memory — OsRng seed | mitigate | D-05: Zeroizing<[u8;32]> field; Drop impl calls zeroize(); never written to disk, registry, or event body; per-session ephemeral only |
| T-84-05 | Repudiation | Tamper-evidence scope — local HMAC only | accept | D-07 ADR (Plan 03 deliverable): explicitly documents tamper boundary = Windows Event Forwarding; in-session HMAC only; cross-session deferred to SEED-005; no overclaiming |
| T-84-06 | Denial of Service | TelemetryConfigInvalid causes abort | mitigate | D-14: telemetry parse failure degrades to defaults, never aborts; confinement enforcement is unaffected by any telemetry failure |
| T-84-SC | Tampering | npm/pip/cargo installs — tracing-etw, eventlog, hmac | mitigate | slopcheck + Task 1 blocking human checkpoint for operator approval before Cargo.toml edits |
</threat_model>
