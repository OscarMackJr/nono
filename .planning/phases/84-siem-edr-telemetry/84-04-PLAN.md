State what the gate proves: SC-1 (Application-log + EventID + JSON named fields), SC-3 (no raw path strings), SC-5 (ETW provider detectable via logman).     CONFIGURATION BLOCK ($script:* variables):
    - $script:EventLogSource = 'nono'
    - $script:EventIdPathDeny = 10001

    - $script:RequiredJsonFields = @('EventType','AgentPid','PathHash','Host','SessionId','ChainHead')

    Assert-True helper: copy verbatim from egress-policy-deny.ps1.

    Helper: Invoke-TriggerDenial — runs a minimal nono command that will produce a path-deny event (e.g. `nono run --profile claude-code -- cmd /c type C:\Windows\System32\config\SAM` which is a known-denied path under the claude-code profile). This seeds the Application log with a fresh event. Returns $true on success (event seeded), $false if nono is not on PATH or run fails. Used in Invoke-Gate before asserting the log entry.

    Test-Precondition:
    1. Admin check: Application-log Event Source registration requires elevated access for querying. Return SKIP_HOST_UNAVAILABLE if not admin.
    2. nono Application-log source registration check: query `Get-WinEvent -FilterHashtable @{LogName='Application';ProviderName='nono';MaxEvents=1} -ErrorAction SilentlyContinue`. If $null AND nono is not on PATH, return SKIP_HOST_UNAVAILABLE with reason "nono Application Event Log source not registered and nono not on PATH — install nono MSI or run nono to register source".
    3. Return $null if all conditions met.

    Invoke-Gate: three sequential assertions:

    SC-1: Application-log entry assertion.
    - Trigger a fresh denial: call `Invoke-TriggerDenial`. If it fails, try to find an existing recent event (last 5 minutes) before failing.
    - Query: `$events = Get-WinEvent -FilterHashtable @{LogName='Application'; ProviderName='nono'; StartTime=(Get-Date).AddMinutes(-5)} -ErrorAction SilentlyContinue | Where-Object { $_.Id -ge 10001 -and $_.Id -le 10005 }`
    - Assert: `$events.Count -gt 0` — else FAIL with "no nono security events found in Application log (EventID 10001-10005) in the last 5 minutes"
    - Parse body: `$body = ($events | Select-Object -First 1).Message`
    - Parse JSON: `try { $parsed = $body | ConvertFrom-Json } catch { FAIL "event body is not valid JSON: ..." }`
    - Assert all SC-1 named fields present: `foreach ($field in $script:RequiredJsonFields) { Assert-True ($null -ne $parsed.$field) "SC-1: field $field missing from event body" }`

    SC-3: No raw path strings in event body.
    - Assert: `-not ($body -match 'C:\\Users\\[^"]+')` — if match found, FAIL "SC-3 FAILED: raw Windows user path found in event body — secret/path leakage (Pitfall 11)"
    - Assert: `-not ($body -match '[A-Za-z]:\\\\[A-Za-z][A-Za-z0-9]*\\\\[A-Za-z0-9_.-]{4,}')` — broader pattern catching raw Windows paths (not hex path hashes)
    - The PathHash field itself is expected (a hex string like "3fa8c12b...") — ensure the check greps for raw path STRINGS not hash values.

    SC-5: ETW provider assertion.
    - Run: `$etw = logman query providers 2>&1 | Select-String -Pattern 'nono' -Quiet`
    - If `$etw` is $false: FAIL with reason "SC-5 FAILED: ETW provider 'nono' not detectable via logman — tracing-etw LayerBuilder registration may have failed or no active trace session exists"
    - Note: `logman query providers` lists registered ETW providers; the tracing-etw LayerBuilder::new("nono") registers the provider on first use. If nono has been run and produced events, the provider should be listed.

    Verdict:
    - All three assertions pass → PASS with detail block showing event count, EventID, body excerpt (first 200 chars), ETW detected
    - Any assertion fails → FAIL with specific assertion name and detail
    - Any SKIP condition → SKIP_HOST_UNAVAILABLE with reason

    NEVER call exit inside the gate. NEVER call Persist-Verdict. Return exactly one verdict [ordered]@{...} object per execution path.
