| CLI arg → AllowDomainEntry | `--allow-domain https://attacker.com/../sensitive` enters via clap; must be URL-parsed + normalized before reaching enforcement |
| Profile JSON → AllowDomainEntry | Untrusted profile files (user-editable) parsed via `#[serde(untagged)]`; empty domain is a failure vector |
| domain string → RouteConfig prefix | `_ep_{domain}` synthetic prefix must not collide or be injectable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01 | Tampering | AllowDomainEntry::WithEndpoints empty domain | mitigate | partition_allow_domain returns Err(ConfigParse) for empty domain — enforced per D-09 fail-secure rule |
| T-56-02 | Tampering | serde(untagged) misparse — a string that looks like an object | accept | #[serde(untagged)] tries Plain first for strings, then WithEndpoints for objects; ambiguity is structurally impossible with untagged |
| T-56-03 | Spoofing | D-12 bare entry widens a scoped host | accept | WATCH-ITEM (D-12): architecture decision + nono why --host output must make effective openness visible; documented in CONTEXT.md |
| T-56-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages introduced — globset + urlencoding already in workspace (Package Legitimacy Audit in RESEARCH.md confirms both Approved) |
