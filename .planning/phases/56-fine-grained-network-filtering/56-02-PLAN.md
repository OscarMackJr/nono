## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI --allow-domain arg → parse_allow_domain_arg | User-controlled URL string; url::Url::parse normalizes percent-encoding + removes .. at parse time (double-safe with normalize_path at match time) |
| endpoint_routes added to resolved.routes | Endpoint routes bypass CONNECT and enter reverse-proxy path; must not expose credential injection path (credential_key: None is enforced) |
| C5 rider plain_hosts.push | Adds endpoint route host:port to the proxy filter allowlist; a malformed upstream: field could push unexpected hosts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-04 | Tampering | parse_allow_domain_arg path traversal via ../  | mitigate | url::Url::parse normalizes .. at parse time; path arriving in EndpointRule.path is already normalized before normalize_path runs at match time — double-safe per D-09 |
| T-56-05 | Spoofing | credential_key: None on endpoint routes | mitigate | partition_allow_domain always sets credential_key: None — endpoint routes carry no credentials; SC2 ordering is structurally preserved (endpoint check at reverse.rs line 96, credential lookup line 119) |
| T-56-06 | Elevation | C5 rider adds unexpected host to filter allowlist | accept | The rider strips only "https://"/"http://" prefixes from the RouteConfig.upstream which was constructed from the operator-supplied domain — no new attack surface beyond what the operator explicitly configured |
| T-56-07 | Information Disclosure | domain_endpoints persisted to capability state file | accept | State file is operator-controlled; endpoint rules are configuration, not secrets; no credential material |
</threat_model>
