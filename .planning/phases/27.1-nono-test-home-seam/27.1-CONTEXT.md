# Phase 27.1: NONO_TEST_HOME Seam - Context

**Gathered:** 2026-05-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a `NONO_TEST_HOME` env-var override to home-directory resolution in `crates/nono-cli/src/` so Windows integration tests can redirect HOME without `LOCALAPPDATA`/`USERPROFILE` drift. Unblocks REQ-AAH-01 (Phase 27 audit-attestation hardening) and queued v2.3 plans (Plan 26-02 PKGS streaming + auto-pull) for Windows-host execution. Closes Phase 27 Blocker 1 (`dirs::home_dir()` not env-overridable on Windows) and Blocker 2 (audit-vs-rollback path mismatch under partial env redirection). Blocker 3 (audit-integrity exit-cleanup `Session not found`) is independent and may surface during Phase 27 test re-enablement — handle in scope if minor, otherwise document and surface as v2.4 follow-up.

**Locked from upstream sources (treat as SPEC-equivalent):**
- Env var name: `NONO_TEST_HOME` (ROADMAP.md:144).
- Production-path no-op when unset; security-equivalent to status quo (ROADMAP.md:156, success criterion #3).
- Scope confined to `crates/nono-cli/src/`; `crates/nono/` stays byte-identical (D-19 cross-phase invariant).
- Resumption path (a) from Phase 27 SUMMARY: re-run Phase 27 redesigned Test 1 body on Windows host with the seam set.

</domain>

<decisions>
## Implementation Decisions

### Helper architecture & callsite scope

- **D-27.1-01:** Add a single `nono_home_dir()` helper in `crates/nono-cli/src/` (module location decided at plan time — likely `crates/nono-cli/src/home.rs` or extension of `config/mod.rs`). Helper checks `NONO_TEST_HOME` first; on hit, validates the path is absolute (matches `validated_home()` rules at `config/mod.rs:32-43`) and returns it; on miss, falls through to the appropriate platform default for the call type (see D-27.1-02).
- **D-27.1-02:** Replace **all 15 home-dir callsites** in `crates/nono-cli/src/` with the new helper:
  - **10 `dirs::home_dir()` callsites:** `audit_commands.rs:725`, `audit_session.rs:36`, `protected_paths.rs:43`, `rollback_commands.rs:245`, `rollback_commands.rs:1520`, `rollback_runtime.rs:202`, `rollback_session.rs:40`, `session.rs:410`, `trust_scan.rs:482`, `trust_scan.rs:486`.
  - **5 `xdg_home::home_dir()` callsites:** `hooks.rs:73`, `package_cmd.rs:175`, `package_cmd.rs:1012`, `package_cmd.rs:1015`, `profile/mod.rs:1979`.
- **D-27.1-03:** Rationale for wrapping `xdg_home` too: today `dirs::home_dir()` and `xdg_home::home_dir()` resolve to **different paths on Windows** (Win32 API vs `USERPROFILE` env var). Without consistent override, the seam creates a partial-redirection bug class identical to Phase 27 Blocker 2 inside the test harness. One helper = one source of truth.
- **D-27.1-04:** `xdg_home` dependency may stay in `Cargo.toml` if other transitive uses remain; otherwise deletion is in-scope housekeeping for the plan to assess.

### Companion seam for Windows path-mismatch (Blocker 2)

- **D-27.1-05:** When `NONO_TEST_HOME` is set, `config::user_state_dir()` on Windows must also redirect, so `audit_root()` and `rollback_root()` co-locate under one root. Mechanics decided at plan time, but must satisfy: (a) audit + rollback land under the same parent when `NONO_TEST_HOME` is set; (b) `user_state_dir()` returns `None` semantics preserved on platforms where it returns None today (no behavior change when unset).
- **D-27.1-06:** Layout under `NONO_TEST_HOME` is the plan's call. Recommended: mirror the production layout — `<NONO_TEST_HOME>/.nono/audit` and `<NONO_TEST_HOME>/.nono/rollbacks` on all platforms (the `~/.nono/...` shape `dirs::home_dir()` callsites already use). The plan should NOT invent a new directory hierarchy — tests want predictable, production-mirroring paths.
- **D-27.1-07:** Other `dirs::*_dir()`-derived knobs (`config_dir`, `data_local_dir`, etc.) are NOT in scope unless surfaced by the Phase 27 test re-enablement (D-27.1-12). The seam covers the audit/rollback co-location problem, not a general home-dir refactor.

### Gating posture & audit-trail

- **D-27.1-08:** **Always-on production seam.** Override compiles into release binaries; no `#[cfg(any(test, debug_assertions))]` gating. Rationale: integration tests that spawn the actual `nono` binary need the override active in whichever build target the test harness uses, and `make ci` includes both debug and release builds. cfg-gating would leave fragile build-mode coupling.
- **D-27.1-09:** **`tracing::warn!` emitted on first use** of the override. Format suggestion: `NONO_TEST_HOME override active: {path}` (final wording at plan time). Rationale: env-var poisoning already a risk vector (see `validated_home()` rationale at `config/mod.rs:23-31`); the warn provides a forensic mark in logs without changing security model. Emit ONCE per process to avoid log spam — `OnceLock<()>` or equivalent guard.
- **D-27.1-10:** **Path validation matches `validated_home()`:** `NONO_TEST_HOME` must be absolute. If set but not absolute, return `NonoError::EnvVarValidation { var: "NONO_TEST_HOME", reason: "must be an absolute path, got: {value}" }` — the error type already exists. Do NOT silently fall through to platform default; fail-closed per the codebase's security principles.
- **D-27.1-11:** No "creation" semantics on the override. The seam returns the path; existing callers handle directory creation as today (`fs::create_dir_all` at the consumer side). Avoid hidden side effects in the helper.

### Phase 27 test re-enablement (in-scope E2E proof)

- **D-27.1-12:** **Phase 27.1 includes re-enabling the Phase 27 audit-attestation tests** (`audit_verify_reports_signed_attestation_with_pinned_public_key` + `rollback_signed_session_verifies_from_audit_dir_bundle`) and adapting their bodies to use `NONO_TEST_HOME`. The seam without an end-to-end proof is dead code. Closes REQ-AAH-01 in this phase. Mirrors Phase 27 SUMMARY resumption path (a).
- **D-27.1-13:** Adaptation steps for the Phase 27 tests:
  1. Remove the `#[ignore]` attributes from both tests (lines 290 and 456 of `crates/nono-cli/tests/audit_attestation.rs`).
  2. Replace the Windows-specific set-difference / `audit_root_for_supervisor()` workarounds in the redesigned Test 1 body (commit `16bae9ca`) with `NONO_TEST_HOME`-driven isolation. Test sets `NONO_TEST_HOME` to a `tempfile::TempDir`, supervisor honors it, audit + rollback land under that root.
  3. Apply the same pattern to Test 2 (`rollback_signed_session_verifies_from_audit_dir_bundle`) — its body was queued behind Test 1 verification per D-AAH-02.
  4. Verify both tests pass under `cargo test -p nono-cli --test audit_attestation` on the Windows dev host with no `#[ignore]` remaining.
- **D-27.1-14:** **Blocker 3 contingency:** If the audit-integrity exit-cleanup `Session not found` issue (Phase 27 SUMMARY) resurfaces with the seam in place, the plan handles it as follows:
  - **If localized fix is small** (e.g., a session-discovery path that uses `dirs::home_dir()` inconsistently and was missed by the seam): fix in scope.
  - **If non-trivial:** Document in plan SUMMARY, surface as v2.4 follow-up, mark the affected test with a temporary `#[ignore]` carrying a Phase 27.1-Blocker-3 note. Do NOT block the seam landing on Blocker 3.
- **D-27.1-15:** Add at least one **dedicated unit test** for the `nono_home_dir()` helper itself (separate from the audit-attestation tests): cases include override set + valid → returned; override set + relative → error; override unset → falls through; on Windows, override changes both `nono_home_dir()` and `user_state_dir()` outputs. This locks the seam contract independent of the integration tests.

### Claude's Discretion

- **Module placement** of the helper (`config/mod.rs` extension vs new `home.rs` file) is the plan's call. Both are acceptable; the existing `config/` module is already the home-resolution authority via `validated_home()` and `user_state_dir()`, so extending `config/mod.rs` is the lower-friction path.
- **Helper return type** (`Result<PathBuf>` vs `Option<PathBuf>`) tracks the call sites: `dirs::home_dir() -> Option<PathBuf>` and most callsites already `.ok_or(NonoError::HomeNotFound)?`, so a `Result<PathBuf>` helper that pre-bakes the error is cleaner. Decided at plan time.
- **Whether to delete `xdg_home` dep** if all callsites migrate is left to the executor; if the dep is otherwise unused after the migration, removing it is in-scope housekeeping.
- **Exact `tracing::warn!` message text** is the plan's call. Must include the resolved path; should not log when the env var is unset (no spam on every run).
- **Test ordering / parallel-safety** for the new helper unit tests: the `validated_home()` ENV_LOCK pattern at `crates/nono-cli/src/test_env.rs` is the established mutex for env-var tests in this codebase — reuse it for the new tests.

</decisions>

## Existing Code Insights

### Reusable Assets
- **`config::validated_home()`** at `crates/nono-cli/src/config/mod.rs:32` — already implements absolute-path validation + structured `NonoError::EnvVarValidation` error for HOME/USERPROFILE/HOMEDRIVE+HOMEPATH chain. The `NONO_TEST_HOME` validator should mirror this pattern (or even share a private helper).
- **`config::user_state_dir()`** at `crates/nono-cli/src/config/mod.rs:106` — extension point for D-27.1-05 (Windows path-mismatch seam). When `NONO_TEST_HOME` is set, this function returns the override-derived state dir.
- **`crate::test_env::ENV_LOCK`** mutex (referenced from `config/mod.rs:113`) — established parallel-safe env-mutation pattern for tests; reuse for new helper unit tests.
- **`NonoError::HomeNotFound`** + **`NonoError::EnvVarValidation { var, reason }`** — both already defined in `crates/nono/src/error.rs`. No new error variants needed for 27.1.

### Established Patterns
- **15 callsites already use `.ok_or(NonoError::HomeNotFound)?`** — the `Result<PathBuf>` helper signature pre-bakes this; migration is mechanical replacement of `dirs::home_dir().ok_or(NonoError::HomeNotFound)?` with `nono_home_dir()?`.
- **Env-var validation pattern:** existing `validated_home()` returns `NonoError::EnvVarValidation { var, reason }` for non-absolute or unset cases. `NONO_TEST_HOME` validation must follow the same shape.
- **Single-emit log pattern:** the codebase uses `OnceLock<()>` / `static AtomicBool` patterns for once-per-process emissions (search the codebase for the closest analog at plan time).
- **Test env-mutation pattern:** `crates/nono-cli/src/env_vars.rs` (per Phase 27 SUMMARY's reference at line 64) already has the set-difference + env-redirection idiom. Pattern reference, not direct reuse.

### Integration Points
- **Phase 27 audit_attestation tests** (`crates/nono-cli/tests/audit_attestation.rs`) — un-#[ignore]'d + adapted to use `NONO_TEST_HOME` (D-27.1-12, D-27.1-13). This is the seam's E2E proof.
- **Phase 26 Plan 26-02** (`auto_pull_loads_registry_pack_extends` test) — listed in 26-CONTEXT.md as blocked on this seam. Not modified by 27.1, but unblocked downstream.
- **`make ci`** must remain green: clippy + fmt + workspace tests. New helper has unit tests; new env-var-using tests use ENV_LOCK; Phase 27 tests pass on Windows host.
