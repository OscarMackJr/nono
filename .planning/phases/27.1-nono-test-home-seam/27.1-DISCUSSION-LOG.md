## Helper scope & callsite coverage

| Option | Description | Selected |
|--------|-------------|----------|
| Wrap all three resolution paths via one helper | Single nono_home_dir() helper; replace all 10 dirs::home_dir() AND 5 xdg_home::home_dir() callsites. One source of truth; consistent under the seam. ~15 sites changed; kills dirs/xdg_home divergence. | ✓ |
| Only dirs::home_dir() callsites (literal ROADMAP) | Replace just the 10 dirs::home_dir() callsites; xdg_home unchanged because it already honors USERPROFILE on Windows. Smaller diff; matches ROADMAP wording but creates partial-redirection bug class. | |

**User's choice:** Wrap all three resolution paths via one helper.
**Notes:** dirs::home_dir() and xdg_home::home_dir() resolve to different paths on Windows (Win32 API vs USERPROFILE env var). Without consistent override, the seam re-creates Phase 27 Blocker 2 internally. One helper = one source of truth.

| Option | Description | Selected |
|--------|-------------|----------|

**Notes:** Phase 27 SUMMARY explicitly identified Blocker 2 (audit-vs-rollback path mismatch); the SUMMARY's resumption path option (b) literally proposes the seam fix Blockers 1 + 2 atomically. One env var, predictable production-mirroring layout.

## Gating posture & audit-trail

| Option | Description | Selected |
|--------|-------------|----------|
| Always-on + tracing::warn at startup when honored | Production binary honors NONO_TEST_HOME if set; emits tracing::warn! at first use. Audit-trail visibility; works in CI release builds; security-acceptable per fail-closed/audit-trail principles. | ✓ |
| #[cfg(any(test, debug_assertions))] gated | Override only compiles in test/debug builds. Zero release-binary surface. Cost: integration tests spawning the actual nono binary may need debug builds; make ci builds release artifacts that would ignore the override. | |
| Always-on, silent (no warn) | Simplest. Honor NONO_TEST_HOME unconditionally with absolute-path validation, no log emission. Cost: no audit trail. | |

**User's choice:** Always-on + tracing::warn at startup when honored.
**Notes:** Path validation must match validated_home() rules (absolute path required; structured EnvVarValidation error on failure). Single emit per process to avoid log spam.
