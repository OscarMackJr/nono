### Edit 1.8 — `crates/nono-cli/src/session.rs` line 410 (in `sessions_dir()`) — DEVIATION (preserves ConfigParse error)

Before:
```rust
pub fn sessions_dir() -> Result<PathBuf> {
    let home = dirs::home_dir().ok_or_else(|| {
        NonoError::ConfigParse("Cannot determine home directory for session registry".to_string())
    })?;
    Ok(home.join(".nono").join("sessions"))
}
```
After:
```rust
pub fn sessions_dir() -> Result<PathBuf> {
    let home = crate::config::nono_home_dir().map_err(|_| {
        NonoError::ConfigParse("Cannot determine home directory for session registry".to_string())
    })?;
    Ok(home.join(".nono").join("sessions"))
}
```
**Rationale (deviation explicitly noted per PATTERNS.md):** This callsite intentionally preserves the contextual `NonoError::ConfigParse` error message. The `.map_err(|_| ...)` discards the underlying `EnvVarValidation`/`HomeNotFound` discriminant from `nono_home_dir()` and substitutes the legacy error string — this matches the pre-migration behavior bit-for-bit for any caller (or test) that matches on the error message. NOTE: this means a non-absolute `NONO_TEST_HOME` will be reported as `ConfigParse("Cannot determine home directory for session registry")` from this callsite, NOT as `EnvVarValidation`. That is intentional per PATTERNS.md guidance ("Decision deferred to plan time" → resolved here as: preserve legacy error message; the dedicated unit tests in Plan 01 cover the `EnvVarValidation` shape directly).
