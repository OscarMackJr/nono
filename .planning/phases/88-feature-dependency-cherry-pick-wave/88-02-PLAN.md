<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LOCALAPPDATA env var → state path | On Windows, the state root derives from %LOCALAPPDATA%; an adversarial env can redirect audit/session state to an attacker-controlled location |
| XDG_STATE_HOME → state path | On Unix, XDG_STATE_HOME override is trusted; must be validated as an absolute path |
| NONO_CONFIG env var → config path | Config directory override; must be validated as an absolute path to prevent relative-path traversal |
| Legacy migration (move) → state root | Atomic move of ~/.nono/ to XDG location; failure must abort (fail-secure) to prevent state split |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-05 | Tampering | state_paths::user_state_dir() Windows arm (%LOCALAPPDATA%) | mitigate | Validate %LOCALAPPDATA% is non-empty; return Setup error if missing or empty (fail-secure; do NOT fall back to a default path) |
| T-88-06 | Tampering | XDG_STATE_HOME override path | mitigate | Upstream state_paths.rs validates the path is absolute; verify this check survives the fork cherry-pick; EnvVarValidation error on relative path |
| T-88-07 | Tampering | NONO_CONFIG override path | mitigate | Validate as absolute path (same pattern as validated_home in config/mod.rs); absolute-path guard enforced at parse time |
| T-88-08 | Denial of Service | Migration failure (D-03) | mitigate | maybe_migrate_legacy_audit_ledger() errors propagate via `?` as fatal; do NOT swallow with .ok() or .unwrap_or_default() — state split is a security integrity failure |
| T-88-09 | Information Disclosure | Dual-read fallback to legacy ~/.nono/ | accept | Legacy fallback is read-only for existing data; new writes go to XDG location; no credential data at rest in these paths (keyring handles secrets) |
| T-88-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
