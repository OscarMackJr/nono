<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| profile JSON → set_vars validation | Untrusted profile data crosses into env injection; validation at parse time is the enforcement boundary |
| env var NONO_KEYRING_TIMEOUT_SECS | Operator-controlled env var crosses into keyring timeout; invalid values must degrade to safe default |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-01 | Tampering | set_vars env injection (exec_strategy.rs) | mitigate | validate_set_vars() rejects PATH and NONO_* prefix; called at profile-load-time before any execve; push_set_vars() deduplicates to prevent shadow attack |
| T-88-02 | Elevation of Privilege | set_vars PATH injection | mitigate | key == "PATH" exact check in validate_set_vars(); LD_PRELOAD and other dangerous vars are intentionally NOT blocked (explicit operator intent per RESEARCH.md) — this is a policy choice, not a mitigation gap |
| T-88-03 | Denial of Service | NONO_KEYRING_TIMEOUT_SECS=0 (infinite wait) | accept | Only operator can set this env var; infinite wait is an explicit opt-in (0 = no timeout per spec); rationale: matches pre-existing behavior when env var is absent |
| T-88-04 | Spoofing | keyring_timeout_secs invalid-value fallback | mitigate | Invalid values fallback to 120s with tracing::warn! (not silent); attackers cannot use an invalid value to disable the timeout since invalid→120s |
| T-88-SC | Tampering | cargo dependency installs | accept | No new packages in this plan; only code changes to existing files |
</threat_model>
