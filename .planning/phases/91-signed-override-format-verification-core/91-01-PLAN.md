<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted token JSON → override.rs parser | Attacker-controlled JSON bytes (key order, whitespace, extra fields, malformed leaves) cross into the canonicalizer |
| received bytes → canonical digest | The digest the signature is checked against MUST derive from the parsed struct, never from the raw received bytes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-01-CANON | Tampering | canonical_bytes() re-derivation | mitigate | Re-derive from parsed Value with BTreeMap code-point sort + R10 strip; NEVER `serde_json::to_string`/`sha256(raw)`. Proven byte-identical against 9 ZT vectors before any signature path exists (PITFALLS #3). |
| T-91-01-UNKNOWN | Tampering | OverrideToken serde model | mitigate | `#[serde(deny_unknown_fields)]` — an injected unsigned field cannot ride alongside the signed object; unknown field → Err{Parse} (fail-secure; OVR-02). |
| T-91-01-MISSING | Spoofing | required signed fields | mitigate | Required OVR-01 fields modeled non-`Option`; absence → Err{MissingField}; `kms_signature.key_id` required non-empty (fail-closed for the VFY-04 ARN check). |
| T-91-01-FLOAT | Tampering | canonicalize leaf validation | mitigate | Reject float/null/bool leaves, out-of-range integers, ctrl/U+2028/U+2029/supra-BMP string chars per CAF R7.4 → Err{Parse} (a malformed leaf cannot silently alter the digest). |
| T-91-01-WIRESHAPE | Tampering | nono-side token construct (D-06) | accept (v1) | The wire shape is a nono-side construct signed by the LOCAL TEST KEYPAIR; reconciliation with real KMS tokens is `[BLOCKING]`-flagged for Phase 93. Deliberate v1 boundary, tracked, not a silent gap. |
| T-91-01-SC | Tampering | cargo dep promotions | mitigate | Only promotions of deps already resolved in `nono-py/Cargo.lock` (base64/chrono/sha2); zero net-new packages → no registry-hallucination/slop surface (RESEARCH Package Legitimacy Audit). Lockfile presence verified before adding. |
</threat_model>
