## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted token + caller-supplied pubkey/ARN allowlist → verify_override | The token is attacker-controlled; the pubkey DER and ARN allowlist come from trusted machine policy (test-injected this phase) |
| crypto-valid signature → authorization | A cryptographically valid signature from a non-allowlisted key MUST NOT authorize |
| offline verify → (Phase 93) live AND-gate | The verified value, not the raw token, crosses into the live check (TOCTOU boundary) |
