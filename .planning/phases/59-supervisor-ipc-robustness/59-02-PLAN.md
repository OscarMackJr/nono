## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandboxed child → supervisor IPC (socketpair fd) | Untrusted child controls frame timing/content; the read side is the attack surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-01a | Denial of Service | macOS `run_supervisor_loop` read of a partial frame | mitigate | Wire `set_read_timeout(5s)`; `read_exact` returns `WouldBlock`/`TimedOut` instead of blocking forever. Slowloris/partial-frame hang bounded; verified by `bounded_read_timeout` test (Task 2). |
| T-59-01b | Denial of Service | reconnect thrash (child loops connect/disconnect) | mitigate | Re-accept is bounded by child liveness (`waitpid(WNOHANG)` already in the loop) + supervisor exit — reconnect cannot outlive the child; no infinite tight loop. |
| T-59-01c | Elevation / Tampering | timeout-induced fail-open | mitigate | A timed-out/partial frame is ABANDONED (fail-closed): no capability is granted on a partial/timed-out request; supervision continues with the sandbox intact (Security Domain invariant). |
| T-59-01d | Tampering | keep-alive leaking seccomp/proxy break-conditions | mitigate | Keep-alive scoped to the URL-open/direct-IPC listener ONLY; existing seccomp/proxy/PTY demotion logic untouched (D-04 anti-pattern). |
| T-59-01e | Spoofing | reconnect impersonation (different principal after re-accept) | mitigate | Existing `SO_PEERCRED`/`peer_credentials` check (socket.rs:380) re-runs on each accept; trust is not cached across reconnect (V3). The socketpair fd is inherited (not filesystem-reachable), bounding the impersonation surface. |
| T-59-SC | Tampering | npm/pip/cargo installs | accept | No package installs — RESEARCH.md §"Package Legitimacy Audit": zero new deps (vendored `libc`, `std` only). No `[ASSUMED]`/`[SUS]` packages → no legitimacy checkpoint. |
