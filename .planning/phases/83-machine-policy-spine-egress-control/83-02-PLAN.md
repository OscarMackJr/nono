<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HKLM policy → daemon startup → two enforcement layers | One read must feed both layers; two independent reads would allow drift |
| Confined agent (AppContainer SID) → network | Agent must have exactly one egress path (loopback→proxy); any direct bypass must be kernel-blocked |
| Low-IL agent → control IPC | Agent must not be able to alter the WFP policy via the control pipe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-83-layer-drift | Tampering / Spoofing | single-source hand-off | mitigate (block-level) | One `read_machine_egress_policy()` at startup feeds ProxyFilter directly AND drives the WFP request; the WFP service NEVER reads HKLM (D-04). |
| T-83-proxy-bypass | Bypass / Spoofing | per-SID WFP filter | mitigate (block-level) | proxy-only + localhost_ports=[proxy_port] => PERMIT loopback:proxy_port (wt 100) + BLOCK all (wt 0). Direct SID→out-of-list-IP is kernel-blocked (D-01/D-02). |
| T-83-peruser-widen | EoP | override resolution | mitigate | Wholesale override (D-08): present policy ignores per-user allow_domain entirely; per-user cannot widen the fleet allowlist. |
| T-83-malformed-failopen | Tampering / EoP | startup read | mitigate (block-level) | `?` propagation aborts startup on PolicyLoadFailed; no `.ok()`/`.unwrap_or` (Pitfall 3). |
| T-83-lowil-ipc | EoP | control pipe | accept (existing) | Control pipe Medium-IL SACL `S:(ML;;NW;;;ME)` already bars Low-IL/AppContainer callers (control_loop.rs); reused, not re-derived. |
</threat_model>
