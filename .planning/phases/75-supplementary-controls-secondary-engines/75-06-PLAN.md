<objective>
Fix GAP-75-A: `nono daemon start` exits 5 (ACCESS_DENIED) when a USER_OWN_PROCESS TEMPLATE (type 50) service is registered, because `sc start` cannot start a template directly.

Purpose: The daemon must reach RUNNING state unconditionally when invoked. The existing `daemon_start_raw_spawn` code path is already proven correct — this plan adds a type-50 detection guard so the template path falls through to the raw-spawn path instead of attempting `sc start`.

Output: Modified `daemon_start()` in `agent_cli.rs` with type-50 guard; unit tests for the three branch-selection paths.
</objective>

Shell idiom to detect type-50 from `sc qc` stdout (string `sc_qc_stdout`):
- The type-50 indicator is the string "50" appearing on the TYPE line, or equivalently the phrase "USER_OWN_PROCESS" (present on type-50 AND type-10 user-own; use "50" specifically, or check for absence of "10" + presence of "USER_OWN_PROCESS").

- Alternative: check `sc_qc_stdout.contains(": 50")` (the exact whitespace-padded form is "TYPE               : 50  USER_OWN_PROCESS").
