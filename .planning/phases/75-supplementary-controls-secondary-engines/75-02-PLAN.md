<objective>
Implement the SUPP-01 demote verb: `nono agent demote <tenant_id>` applies a post-hoc IL-drop to a running daemon-launched agent (incident-response lever only; never the primary confinement boundary).

Purpose: An operator can lower a misbehaving agent's token integrity level on the fly as a kill-switch lever. D-03 planning call: demote ALSO cuts the per-agent WFP filter (severs egress at the same time, since leaving egress open after IL-drop is a security regression). Spike-002 leak limits are documented inline as the soundness boundary.

Output: `ControlRequest::Demote` + `handle_demote` in `control_loop.rs`; `AgentCommands::Demote` in `cli.rs`; `agent_demote` in `agent_cli.rs`. Unit tests covering the demote-unknown-tenant error, parse test, and (cross-platform) structural test for the IL-drop path.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → control pipe → daemon | nono agent demote sends a JSON request over \\.\pipe\nono-agentd-control. The pipe has Medium-IL minimum SDDL (from Phase 74 T-74-07-01): Low-IL / AppContainer agents cannot open it. Only a Medium-IL operator can invoke demote. |
| handle_demote → tenant's process | The daemon holds the OwnedHandle (from launch_agent) and DuplicateHandle before IL-drop calls. The tenant_id is validated against DaemonState::tenants before any Win32 call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-75-02-01 | Spoofing | Agent self-demotes via nono agent demote | mitigate | Control pipe SDDL requires Medium IL minimum (T-74-07-01 carry-forward). AppContainer/Low-IL agents cannot open the control pipe. Demote can only be issued by the interactive Medium-IL operator. |
| T-75-02-02 | Tampering | Tenant A sends demote request with Tenant B's tenant_id | mitigate | handle_demote validates tenant_id against DaemonState::tenants (operator-visible list via nono agent list only). Tenant IDs are 32-char random hex strings — computationally infeasible to guess. |
| T-75-02-03 | Denial of Service | Demote unexpectedly kills the agent's legitimate work | accept | Documented leak limit: IL-drop may cause agent malfunction. Operator accepts this consequence; demote is an incident-response lever. |
| T-75-02-04 | Elevation of Privilege | IL-drop fails but WFP filter still present → agent has reduced IL but full egress | mitigate | handle_demote calls wfp_filter_remove after IL-drop (D-03 WFP-cut); failure of wfp_filter_remove logs warning (best-effort; WFP service startup sweep provides backstop). |
| T-75-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages. Not applicable. |
</threat_model>
