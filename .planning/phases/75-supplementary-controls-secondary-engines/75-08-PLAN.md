<objective>
Fix GAP-75-C (SC3, surfaced by the live UAT 2026-06-16): a daemon-launched Copilot CLI agent
confines correctly (write-outside-workspace denied, AppContainer spawn) but does NOT run to
completion. Copilot is a Node ESM app (A4 = YES); Node's ESM `realpathSync` walks the module path
to the drive root and `lstat('C:\')` is denied under AppContainer (`EPERM`, errno -4048), so the
engine dies during module resolution.

Purpose: make Node-ESM engines (Copilot, and any node-backed CLI) runnable confined, WITHOUT
weakening the confinement boundary. This is the engine-completion half of SC3 — the confinement
half is already proven (the fail-secure launch-coverage gate works; write-outside is denied).

Output: a spike-validated minimal fix (drive-root attribute access for the package SID and/or a
Node resolver mitigation), copilot-cli profile coverage (node.exe interpreter + real package dir
via symlink resolution), applied on BOTH the daemon launch path and the `nono run` path, plus an
SC3 re-run proving copilot completes confined.
</objective>
