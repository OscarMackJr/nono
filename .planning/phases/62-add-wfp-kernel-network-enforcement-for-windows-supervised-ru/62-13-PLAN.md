<objective>
Gap-closure for the 62-12 AppContainer spawn bug, root-caused + SPIKE-VALIDATED in debug wfp-write-restricted-0142.
62-12 only DERIVES the package SID (DeriveAppContainerSidFromAppContainerName); CreateProcessW(SECURITY_CAPABILITIES)
then fails ERROR_FILE_NOT_FOUND because the AppContainer PROFILE is never registered — even from a fully-accessible
cwd (the spike spawned curl from C:\Windows\System32 and still failed until the profile was registered, proving the cwd
was never the spawn blocker). The decisive spike ALSO proved the payoff: with CreateAppContainerProfile called first,
the AppContainer child STARTS and WFP kernel-BLOCKS its outbound connection via the EXISTING ALE_USER_ID(packageSid)
filter (curl: "Could not resolve host").

Fix: the broker registers the per-run AppContainer profile (CreateAppContainerProfile) before the spawn and deletes it
when the child exits (RAII). Keep the existing ALE_USER_ID WFP scoping (validated). Add a best-effort package-SID
FILE_TRAVERSE grant on the user-owned ancestor of the cwd so the confined child can traverse into a profile-deep cwd.

SCOPE: makes the confined AppContainer child START + be WFP-blocked end-to-end in nono (targets a passing SC1 with curl
BLOCKED, not crashed). The full READ-grant model for arbitrary tools that read many user paths (e.g. claude.exe) remains
a follow-up. Whether the lowbox has bypass-traverse (and thus whether ANY ancestor grant is even needed) is confirmed by
the follow-up live UAT. 62-11 (uninstall purge) is independent.

Output: confined --block-net child spawns as a REGISTERED per-run AppContainer, starts cleanly, and its outbound TCP is
kernel-blocked by the package-SID WFP filter.
</objective>

| Boundary | Description |
|----------|-------------|
| broker → AppContainer profile registry/namespace | per-run profile created + deleted (RAII); no persistent profile accumulation |
| AppContainer child → user-owned cwd ancestors | package SID granted FILE_TRAVERSE only (pass-through, not read/enumerate), reverted on Drop |

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-62-30 | Denial of Service (self) | CreateProcessW FILE_NOT_FOUND (unregistered profile) | mitigate | Register the profile (CreateAppContainerProfile) before spawn — spike-proven the child then starts. |
| T-62-31 | Spoofing / silent non-enforcement | spawning into an unregistered/wrong profile | mitigate | FAIL-CLOSED: profile-create failure (not ALREADY_EXISTS) propagates, never spawns; single-source name → same package SID on spawn + WFP filter. |
| T-62-32 | Tampering (leave-behind) | orphaned AppContainer profile | mitigate | RAII DeleteAppContainerProfile on the broker child-exit; ALREADY_EXISTS tolerance recovers a crashed run's orphan. |
| T-62-33 | EoP / over-broad traverse | ancestor FILE_TRAVERSE grant | accept (minimal) | Traverse-only (no read/list), per-run package SID, ONLY user-owned ancestors (non-owned skipped), reverted on Drop. |
