supervised run needs it. Without the SDDL fix, even a running service rejects the non-elevated
caller with "Access is denied" (error 5), making D-01 end-to-end unreachable.

Output: nono-machine.wxs with Start="auto" (+ optional ServiceConfig); nono-wfp-service.rs with
updated PIPE_SDDL and a new test_wfp_pipe_sddl_includes_interactive_users unit test.

SDDL current grants:
  (A;;GA;;;SY) = SYSTEM: full access
  (A;;GA;;;BA) = Administrators: full access
  (A;;GRGW;;;OW) = Owner: generic read + write
  MISSING: Interactive Users (IU) — non-elevated Medium-IL processes hold IU enabled

SDDL target:
  "D:(A;;GA;;;SY)(A;;GA;;;BA)(A;;GRGW;;;IU)(A;;GRGW;;;OW)"

Security note from RESEARCH.md §Security Domain:
  nono.exe runs at Medium IL when initiating the WFP IPC call (broker arm calls nono run at
  Medium IL, not Low IL). Medium-IL processes hold the Interactive Users SID as ENABLED.
  Therefore IU grants are effective. Adding GRGW to IU does NOT grant the confined child
  (Low-IL) access to deactivate its own WFP filters — the Low-IL child does not call nono run
  directly. This is confirmed in A3 of the Assumptions Log.

SDDL conversion machinery (nono-wfp-service.rs lines 573-619): unchanged — the constant is
  converted once per accept-loop iteration via ConvertStringSecurityDescriptorToSecurityDescriptorW.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nono.exe (Medium IL) → \\.\pipe\nono-wfp-control | DACL gate; SDDL controls which callers can connect |
| SCM boot-start → nono-wfp-service (SYSTEM) | Service runs as SYSTEM; pipe is the only external interface |
| MSI uninstall → ServiceControl Remove | WiX Remove="uninstall" is the primary leave-nothing mechanism |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-04 | Elevation of Privilege | PIPE_SDDL IU ACE | mitigate | Grant only GRGW (generic read+write) to IU, NOT GA (full access); SYSTEM and Administrators retain GA; IU cannot create the server pipe (only clients connect); the pipe protocol only activates/deactivates WFP filters and does not expose a "disable all" API. Low-IL sandbox children cannot deactivate their own filters because nono.exe itself (the pipe client) runs at Medium IL |
| T-62-05 | Elevation of Privilege | PIPE_SDDL over-grant risk | mitigate | Use IU (Interactive Users) not WD (World/Everyone) — avoids granting non-interactive service accounts and anonymous callers; confirmed by RESEARCH.md §Open Questions #2 |
| T-62-06 | Denial of Service | start=auto crash loop | mitigate | ServiceConfig single-restart policy (FirstFailureActionType="restart", ResetPeriodInSeconds="60") limits auto-restart storms; third failure = none (service stops, D-03 in nono run handles the fallback gracefully) |
| T-62-07 | Tampering | start=auto + msiexec /x leave-nothing regression | mitigate | ServiceControl Stop="both" Remove="uninstall" unchanged — WiX instructs Windows Installer to stop and remove the service during uninstall regardless of start type; CaUninstallWfpServices is defense-in-depth. Verified in Phase 53 UAT-C for start=demand; start=auto does not change the removal path |
| T-62-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
