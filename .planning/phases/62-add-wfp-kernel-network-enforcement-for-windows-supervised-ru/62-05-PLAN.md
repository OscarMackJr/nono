<objective>
Gap-closure for F-62-01. Plan 62-02 set Start="auto" in the checked-in dist/windows/nono-machine.wxs,
but that file is a GENERATED reference snapshot: both the CI release path (release.yml → build-windows-msi.ps1)
and the local POC path (sign-poc-local.ps1 → build-windows-msi.ps1) regenerate it from an inline here-string
in scripts/build-windows-msi.ps1 whose ServiceInstall template hardcodes Start="demand". As a result, no build
path currently produces an auto-start MSI, and REQ-WFP-01 truth #1 ("the machine MSI registers nono-wfp-service
with start=auto") is unmet. Additionally, scripts/validate-windows-msi-contract.ps1 actively asserts
Start="demand", so it would red-fail any auto generator change.

This plan changes the generator to emit Start="auto", retargets the existing contract guard to expect "auto"
(turning it into the regression guard for the new value), and regenerates the tracked .wxs snapshot so it
matches the generator output.

Purpose: make Start="auto" actually land in the shipped/built MSI so UAT SC1-step-3 (sc qc → AUTO_START),
SC2 (boot-start survives reboot), and REQ-WFP-01 truth #1 are achievable.

Output: build-windows-msi.ps1 with Start="auto"; validate-windows-msi-contract.ps1 asserting "auto";
regenerated dist/windows/nono-machine.wxs snapshot.
</objective>
