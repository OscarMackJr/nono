<objective>
Implement the D-03 "ensure service running or abort fail-closed" decision path in network.rs:
add a third injectable `start_service_fn` parameter to `install_wfp_network_backend_with_runner`,
intercept `BackendServiceStopped` before the fail-closed return, attempt to start the service,
re-probe on success, and return an updated error naming the exact remediation command on failure.
Also align `build_wfp_service_create_args` to `start=auto` (so `nono setup --register-wfp-service`
cannot silently revert the MSI posture), update the stale service description string, and add the
three mock-based unit tests that verify the D-03 decision branches without elevation.

Purpose: D-03 provides defense-in-depth for dev layouts, user-scope installs, and post-crash
scenarios where the boot-start service (D-01) was never started or was stopped. It also preserves
the no-pass-through guarantee that is nono's security invariant.

Output: Modified network.rs with new start_service_fn parameter, D-03 interception block,
updated fail-closed error messages, aligned start=auto in build_wfp_service_create_args,
accurate description string, and 3 new passing unit tests.
</objective>
