<objective>
Gap-closure for F-62-UAT-04 (root-caused in debug session wfp-wrong-session-dynamic). open_wfp_engine opened a
FWPM_SESSION_FLAG_DYNAMIC session, making WFP objects private to the opening session. The service creates the
sublayer NONO_SUBLAYER_GUID in its long-lived startup engine (run_named_pipe_server L562-563), but per-request
install_wfp_policy_filters opens a SEPARATE dynamic engine whose FwpmFilterAdd0 references that sublayer → WFP
rejects the cross-session reference with FWP_E_WRONG_SESSION (0x8032000C). The correct model is PERSISTENT: the
sublayer and filters survive engine-handle close and are referenceable across the startup engine, per-request
install engines, and per-request remove engines. The whole cleanup architecture (startup orphan sweep,
deterministic-key remove, MSI uninstall) only makes sense for persistent objects.

SCOPE (per user sequencing decision): this plan makes ONLY the session-flag change to unblock SC1. The leave-nothing
cleanup gap it exposes (the uninstall path does not yet delete the now-persistent sublayer/filters) is closed by the
follow-up plan 62-11 BEFORE SC4. Do not attempt the uninstall purge here.

Purpose: eliminate FWP_E_WRONG_SESSION so the service's real FwpmFilterAdd0 path installs the block filter and SC1
can pass.

Output: open_wfp_engine uses a persistent session (flags = 0); the unused FWPM_SESSION_FLAG_DYNAMIC import removed.
</objective>
