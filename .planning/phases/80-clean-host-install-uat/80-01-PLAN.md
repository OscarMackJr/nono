<objective>
Apply the two-part build fix (D-03 + D-04) that closes the Phase 67 carry-forward INST-01
failure mode on a clean Win11 host.

Purpose: A clean host without vcruntime140.dll causes msiexec exit 1603 because the nono
binaries fail to load (STATUS_DLL_NOT_FOUND), which triggers a full install rollback. Two
changes prevent this: (a) static CRT linkage eliminates the DLL dependency, and (b) making
the nono-wfp-service start non-fatal prevents a service SCM timeout from rolling back an
otherwise-successful install.

Output: `.cargo/config.toml` (new, local dev convenience), patched `.github/workflows/ci.yml`
and `.github/workflows/release.yml` (step-level RUSTFLAGS for CI/release correctness), patched
`scripts/build-windows-msi.ps1` (Vital="no" on ServiceInstall + ErrorControl="ignore"), and
new Assert-Equal assertions in `scripts/validate-windows-msi-contract.ps1` locking this
non-fatal contract going forward.
</objective>
