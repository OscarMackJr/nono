## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gate script → msiexec.exe | Gate passes MSI path and install flags to msiexec; path must be literal (no shell interpolation) |
| gate script → pwsh.exe (fresh session) | Gate launches a new pwsh process to probe PATH; arguments are static strings, no user input |
| operator → -MsiPath parameter | Operator-supplied path; must be validated with Test-Path -LiteralPath before use in msiexec invocation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-04 | Tampering | `-MsiPath` operator-supplied param used in msiexec invocation | mitigate | Use `Test-Path -LiteralPath $script:MsiPath` (not wildcard); `Start-Process -ArgumentList @('/i', $script:MsiPath, ...)` passes path as an array element (no shell interpolation, no string concatenation) — per RESEARCH.md security domain |
| T-80-05 | Tampering | msiexec log file path injection | mitigate | Log path is `Join-Path $env:TEMP 'nono-gate-install.log'` — fixed suffix, no operator input in the log path — per RESEARCH.md security domain |
| T-80-06 | Spoofing | Dirty-host detection bypass (nono installed to non-standard path) | accept | D-02 is scoped to "has nono been installed via this MSI" — Program Files + service check is sufficient; a missed dirty host produces an incorrect PASS at worst; risk accepted by D-01 operator-VM model (per CONTEXT.md D-02 rationale) |
| T-80-07 | Tampering | Stray pipeline output from Invoke-Gate (WR-01) | mitigate | All Start-Process invocations assign result to a named variable; $null = ... or | Out-Null on any incidental pipeline output; no bare process object written to pipeline |
| T-80-SC | Tampering | No npm/pip/cargo installs introduced this plan | accept | Pure PowerShell; no new packages; no slopcheck needed |
