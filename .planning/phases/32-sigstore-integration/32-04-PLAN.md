<objective>
Close the broker-binary trust loop opened by Phase 31. Reuse Phase 28's `query_authenticode_status` chain-walker to extract `nono.exe`'s OWN Authenticode signature at every broker dispatch, then require `nono-shell-broker.exe`'s signature to match (subject AND thumbprint) before `CreateProcessAsUserW`. Self-bootstrapping (D-32-13): if `nono.exe` runs at all on Windows, its signature is implicitly trusted by the OS — broker trust = "broker is signed by whoever signed me." Fail-closed on mismatch (D-32-12, no escape hatch). Skip verification only in dev-build layouts via an install-layout substring detector (Pitfall 6: `#[cfg(debug_assertions)]` would falsely trigger production-mode strict checks under `cargo test --release`).

Purpose: Phase 31 ships the broker dispatch arm but does no signature check on the broker. An attacker (with write access to `<install_dir>\nono\`) could swap a hostile broker. Authenticode self-trust-anchor closes that gap with ~30 lines of glue at the corrected dispatch site (`launch.rs:1246-1438` — NOT `:2173+` which is the test module per PATTERNS.md anchor correction).

Output: `WindowsTokenArm::BrokerLaunch` arm wraps an Authenticode self-trust-anchor verify around the existing `CreateProcessW` call; 6 Windows-only integration tests cover D-32-11/12/13/14 acceptance.

    ```rust
    // Phase 32 D-32-11/13/14: Authenticode self-trust-anchor.
    // On every dispatch (no cache, D-32-14), require broker.exe's Authenticode
    // signer subject + thumbprint to match nono.exe's own. Fail-closed; no
    // escape hatch (D-32-12). Dev-build skip via install-layout detector
    // (Pitfall 6 — #[cfg(debug_assertions)] would false-trigger under cargo
    // test --release).
    if !is_dev_build_layout(&nono_exe) {
        use crate::exec_identity::AuthenticodeStatus;

        let nono_status = crate::exec_identity_windows::query_authenticode_status(&nono_exe)?;
        let broker_status = crate::exec_identity_windows::query_authenticode_status(&broker_path)?;

        let (nono_subject, nono_thumbprint) = match nono_status {
            AuthenticodeStatus::Valid { signer_subject, thumbprint } => (signer_subject, thumbprint),
            other => return Err(NonoError::TrustVerification {
                path: nono_exe.display().to_string(),
                reason: format!(
                    "nono.exe Authenticode status is {other:?} (expected Valid). \
                     Self-trust-anchor unavailable; refusing to spawn broker."
                ),
            }),
        };
        let (broker_subject, broker_thumbprint) = match broker_status {
            AuthenticodeStatus::Valid { signer_subject, thumbprint } => (signer_subject, thumbprint),
            other => return Err(NonoError::TrustVerification {
                path: broker_path.display().to_string(),
                reason: format!(
                    "broker.exe Authenticode status is {other:?} (expected Valid). \
                     Refusing to spawn."
                ),
            }),
        };
        if nono_subject != broker_subject || nono_thumbprint != broker_thumbprint {
            return Err(NonoError::TrustVerification {
                path: broker_path.display().to_string(),
                reason: format!(
                    "broker.exe Authenticode signature does not match nono.exe — \
                     expected subject `{nono_subject}` thumbprint `{nono_thumbprint}`, \
                     got subject `{broker_subject}` thumbprint `{broker_thumbprint}`. \
                     Refusing to spawn."
                ),
            });
        }
        tracing::debug!(
            target = "broker_authenticode",
            "broker.exe Authenticode self-trust-anchor verified: subject={nono_subject} thumbprint={nono_thumbprint}"
        );
    } else {
        tracing::info!(
            target = "broker_authenticode",
            "skipping broker Authenticode verify: dev-build layout detected at {}",
            nono_exe.display()
        );
    }
    ```

    ```rust
    /// false otherwise. Skipping Authenticode verify in dev avoids failing
    /// `cargo test --release` when broker.exe is unsigned. See Phase 32
    /// `.planning/phases/32-sigstore-integration/32-RESEARCH.md` Pitfall 6.
    fn is_dev_build_layout(nono_exe_path: &std::path::Path) -> bool {
        let s = nono_exe_path.to_string_lossy();
        // Both Windows-style backslashes and Unix-style forward slashes are
        // checked. The Unix paths cover macOS/Linux dev runs of the test

            || s.contains("/target/release/")
    ```
