## Preconditions (must be true before starting)
- Dev-layout or signed nono.exe present (R-B4 broker trust gate)
- nono-agentd.exe built and in PATH (`cargo build --release -p nono-cli --bin nono-agentd`)
- Running from PowerShell (NOT git-bash/MSYS) — supervised runs fail under MSYS due to no real console (memory `feedback_windows_supervised_needs_real_console`)
- Confirmed no pre-existing nono-agentd SCM service (run `sc query nono-agentd` — expected: FAILED 1060 = "The specified service does not exist")
- A test workspace dir owned by the current user exists (e.g. `%USERPROFILE%\nono-test-workspace`) — R-B3 ownership requirement

## SC4 — Privilege Model (run FIRST; blocks the rest if SYSTEM)
Steps:
1. Run: `nono daemon install` (if SCM path) or note the foreground fallback mode
2. Run: `sc qc nono-agentd`
3. Expected output: `TYPE : 110 USER_OWN_PROCESS` (NOT `WIN32_OWN_PROCESS`)
4. Expected: `SERVICE_START_NAME: <current_user_account>` (NOT `LocalSystem`)
5. Pass criterion: ServiceType is USER_OWN_PROCESS; account is NOT LocalSystem

## SC1 — Two Concurrent Agents (the marquee test)
Steps:
1. Start the daemon: `nono daemon start` (or `nono-agentd --foreground` in one terminal)
2. In a second PowerShell window: `nono agent launch --profile aider -- cmd.exe /C "echo Agent A; timeout /t 30; echo Done A"`
3. In a third PowerShell window: `nono agent launch --profile aider -- cmd.exe /C "echo Agent B; timeout /t 30; echo Done B"`
4. In a fourth window: `nono agent list` — expected: shows TWO tenants with distinct tenant IDs and distinct package SIDs (S-1-15-2-... strings)
5. `nono classify <Agent A PID>` → AI_AGENT; `nono classify <Agent B PID>` → AI_AGENT; `nono classify <current PS PID>` → not an agent
6. Let both agents complete (timeout exits); run `nono agent list` again → expected: zero tenants
Pass criteria: (a) two distinct SIDs in `nono agent list`; (b) both agents complete without hanging; (c) tenant list returns to zero after exit

## SC2 — Cross-Tenant Denial (in-process integration test was Wave 0; this confirms at integration level)
Steps:
1. Confirm: `NONO_DAEMON_INTEGRATION_TESTS=1 cargo test -p nono-cli daemon_cross_tenant_denial` → PASS (carried from Wave 0)
2. Manual double-check: with two agents running (A and B), attempt to manually connect `CreateFileW` to agent A's named pipe instance using agent B's token — this requires a custom test tool. If such a tool was written as part of the spike, run it here. If not, the Wave 0 automated test is sufficient for SC2.
Pass criteria: Wave 0 `daemon_cross_tenant_denial` test PASS; record the test output in SUMMARY.md

## SC3 — Handle Baseline (100-agent reap — from Wave 0 spike)

2. Confirm: baseline handle count and post-run handle count in test output; post ≤ baseline + 5
Pass criteria: Test passes; baseline vs post-run delta ≤ 5; record both numbers in SUMMARY.md
