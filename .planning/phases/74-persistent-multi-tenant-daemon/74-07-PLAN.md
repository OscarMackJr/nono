<client_protocol_ALREADY_IMPLEMENTED>
The CLI client (crates/nono-cli/src/agent_cli.rs) ALREADY speaks this protocol — the daemon side MUST match it EXACTLY:
- Pipe name: `\\.\pipe\nono-agentd-control` (const DAEMON_CONTROL_PIPE_NAME in agent_daemon/mod.rs)
- Wire framing: `[4-byte little-endian length][JSON payload UTF-8]` (same as socket_windows.rs framed JSON)
- Launch request:  {"action":"launch","profile":"<name>","cmd":["exe","arg1",...]}
- List request:    {"action":"list"}
- Response: a single framed string the client prints verbatim (`response.trim()`).
  - launch → a human-readable confirmation incl. tenant_id + package SID (and pid).
  - list   → a human-readable tenant listing (e.g. "Tenant agents (N):\n  <id>  profile=..  sid=S-1-15-2-..  pid=..").
- Client opens with GENERIC_READ|GENERIC_WRITE, 5s WaitNamedPipeW timeout, MAX_RESPONSE 64KiB.

launch_agent signature (agent_daemon/launch.rs):

  Resolve `profile` by name from policy.json/builtins (see crate::profile loading); exe = cmd[0], args = cmd[1..].

Document the chosen SDDL with a // SAFETY/security comment.

PER-CONNECTION HANDLER:
1. Read [4-byte LE len][JSON] (reuse the framing read pattern from socket_windows.rs / the client).
2. Parse {"action": "..."}.
3. action == "launch": resolve `profile` (name → nono::profile::Profile via crate::profile loader; reject unknown profile fail-secure with a clear error response). Split `cmd`: exe = PathBuf::from(cmd[0]); args = cmd[1..]. Call launch_agent(daemon_state.clone(), &profile, exe, args).await.
