**Fix 1 — clean detach (agent_cli.rs daemon_start dev-layout spawn):**
When spawning nono-agentd.exe as the dev-layout background daemon, prevent it from holding the parent console's handles:
- `.stdin(Stdio::null()).stdout(Stdio::null()).stderr(Stdio::null())`
- On Windows: `use std::os::windows::process::CommandExt;` and `.creation_flags(DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP)` (DETACHED_PROCESS = 0x0000_0008, CREATE_NEW_PROCESS_GROUP = 0x0000_0200).
This makes the operator's shell return immediately. Keep the printed "started (pid=...)" message. (The daemon's foreground-mode banner should no longer leak into the caller.)

**Fix 2 — bare-exe path resolution (launch.rs, before CreateProcessW):**
Before `spawn_appcontainer_process_suspended` calls CreateProcessW, resolve the exe:
- If the exe path is already absolute AND exists → use as-is.
- Else resolve via `SearchPathW` (Win32, in windows-sys Storage::FileSystem) against the standard search path (pass null lpPath to use the default PATH search; append ".exe" default extension). On success → absolute path.
- If unresolvable → return a CLEAR NonoError: `"agent launch: executable '<name>' not found (provide an absolute path or ensure it is on PATH)"` — NOT a raw os-error-2.
- // SAFETY docs on the SearchPathW unsafe block; unwrap-free. Do NOT weaken any existing exe-coverage/profile validation — resolve FIRST, then any existing coverage check applies to the resolved absolute path. The AppContainer/job confinement is unchanged.

**Fix 3 — engine profile in list (mod.rs + launch.rs + control_loop.rs):**
- Add an `engine_profile: String` field to AgentTenant (the requested engine profile name, e.g. "aider"). Keep the existing AppContainer `profile_name` (nono.session.<id>) for the OS profile.
- In launch_agent, set `engine_profile` from the requested `profile`'s name (the `&Profile` arg / the wire `profile` string).
- In control_loop's list handler, display `profile=<engine_profile>` instead of the AppContainer profile_name. (The launch RESPONSE already shows the engine profile correctly — make list consistent.)

No WFP imports; no escape hatch; no confinement regression. unwrap-free; SAFETY-documented.
