<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nono-agentd binary → SCM | SERVICE_USER_OWN_PROCESS is the privilege gate; using OWN_PROCESS here would make it a machine-wide SYSTEM service (Pitfall 4) |
| AgentTenant::Drop → Windows kernel | KILL_ON_JOB_CLOSE fires when job_handle closes; OwnedHandle::Drop is the chain |
| AgentTenant::Drop → HKCU registry | DeleteAppContainerProfile cleans up; failure is non-fatal (daemon stays up) but accumulates registry entries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-03-01 | Elevation of Privilege | nono-agentd running as SYSTEM (wrong service type) | mitigate | Acceptance criterion: "SERVICE_USER_OWN_PROCESS" string must appear in nono-agentd.rs; reviewer cross-checks against ADR Decision 1 |
| T-74-03-02 | Elevation of Privilege | WFP API imports in nono-agentd | mitigate | Acceptance criterion: "Fwpm" must NOT appear in nono-agentd.rs; ADR Decision 4 enforced |
| T-74-03-03 | Denial of Service | AgentTenant::Drop panics on DeleteAppContainerProfile failure | mitigate | Drop uses tracing::warn! + let _ pattern; must not panic; test verifies graceful handling |
| T-74-03-04 | Information Disclosure | AppContainer profile names accumulating in HKCU | mitigate | DeleteAppContainerProfile in Drop; best-effort (non-fatal on failure) to prevent unbounded accumulation |
| T-74-SC | Tampering | npm/pip/cargo installs | accept | No new packages |
</threat_model>
