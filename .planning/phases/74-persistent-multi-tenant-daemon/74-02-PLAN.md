struct ImpersonationGuard;

Add `pub(crate) fn authenticate_pipe_client(pipe_handle: HANDLE) -> Result<String>` in the `#[cfg(target_os = "windows")]` production code block, placed near the `build_capability_pipe_sddl` and `validate_package_sid_for_sddl` functions. The function:
1. Calls `ImpersonateNamedPipeClient(pipe_handle)`; returns `Err` on failure (GLE 0 → `std::io::Error::last_os_error()`).
2. Creates an `ImpersonationGuard` IMMEDIATELY after the successful impersonation. All subsequent code paths fall through to the guard's `drop`, which calls `RevertToSelf`.
3. Opens the impersonated thread token via `OpenThreadToken(GetCurrentThread(), TOKEN_QUERY, 0, &mut token)`.
4. Calls `GetTokenInformation(token, TokenAppContainerSid_variant, ...)` using the two-pass probe+fill pattern from `current_logon_sid` (lines 1473-1507) — adapt for `TOKEN_APPCONTAINER_INFORMATION` (the struct returned for `TokenAppContainerSid`).
5. Extracts the SID pointer from the result struct and calls `ConvertSidToStringSidW` to get the string form (pattern from lines 1526-1544).
6. Calls `CloseHandle(token)`.
7. Returns `Ok(sid_string)`. The `ImpersonationGuard` drops at end of scope → `RevertToSelf`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticate_pipe_client → ImpersonateNamedPipeClient | Server adopts client's security context; MUST revert on ALL paths (RAII guard) |
| socket_windows.rs → AgentRegistry | The SID returned by authenticate_pipe_client is compared against the registry; if the comparison is wrong-side-equal, cross-tenant theft is possible |
| SupervisorMessage wire format | session_id is a routing hint, never an authz signal; the no-tenant-id-field test guards this invariant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-02-01 | Elevation of Privilege | authenticate_pipe_client: ImpersonateNamedPipeClient without RevertToSelf | mitigate | ImpersonationGuard RAII struct calls RevertToSelf on drop; test `authenticate_pipe_client_reverts_on_error` verifies error-path revert |
| T-74-02-02 | Spoofing | SupervisorMessage wire-frame session_id used as authz signal | mitigate | `supervisor_message_no_tenant_id_field` test; authenticate_pipe_client returns kernel-vouched SID (not wire-frame value); accept-loop uses SID for authz, session_id only as routing hint |
| T-74-02-03 | Information Disclosure | TokenAppContainerSid extraction from impersonated token | mitigate | SID is only returned to the daemon's accept loop (in-process); never logged or transmitted in plaintext; CloseHandle(token) called before returning |
