`pub(crate) fn run_daemon(args: cli::DaemonArgs) -> nono::Result<()>`:
- Match on `args.command`:
  - `DaemonCommands::Start`: on Windows, spawn `nono-agentd.exe` as a foreground process (or call the daemon control pipe if already running); on non-Windows print "nono-agentd is Windows-only" and return Ok(()).
  - `DaemonCommands::Stop`: send a stop request via the daemon control pipe (or `sc stop nono-agentd`). For Phase 74, a simple implementation: `std::process::Command::new("sc").args(["stop", "nono-agentd"]).status()` is acceptable; emit a comment that Phase 75 may wire a proper control pipe.
  - `DaemonCommands::Status`: print whether the daemon is running (check the pipe exists / sc query). On non-Windows: "nono-agentd is Windows-only".
  - `DaemonCommands::Install`: on Windows, `sc create nono-agentd type= userservice start= auto binpath= "C:\..."` or use the SCM Rust API directly. For Phase 74, a brief implementation using `std::process::Command::new("sc")` is acceptable with a TODO for the proper MSI integration in Phase 75.
  - `DaemonCommands::Uninstall`: `sc delete nono-agentd`.

`pub(crate) fn run_agent(args: cli::AgentArgs) -> nono::Result<()>`:
- `AgentCommands::Launch(launch_args)`: on Windows, connect to the daemon's control pipe and send a LaunchAgent request with `profile` and `cmd`. For Phase 74, a minimal implementation: connect to `\\.\pipe\nono-agentd-control` and write a JSON payload `{"action":"launch","profile":"...","cmd":[...]}`. If the daemon is not running, return `Err(NonoError::SandboxInit("nono-agentd not running; use 'nono daemon start' first".into()))`. On non-Windows: error.
- `AgentCommands::List`: connect to the daemon control pipe and send `{"action":"list"}`; print the response tenant list. Phase 74 minimal: if pipe not available, print "No daemon running".

The control pipe name used by `agent_cli.rs` must match the name used in the daemon's `run_accept_loop` — derive it as `\\.\pipe\nono-agentd-control`. Define it as a shared constant in `agent_daemon/mod.rs` and import it here.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI verb surface → daemon control pipe | The control pipe is the trust boundary; agent_cli.rs connects to it as an untrusted client |
| `nono agent launch` cmd arg | User-supplied command; passed to the daemon which validates the profile before launching |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-05-01 | Spoofing | agent_cli sends LaunchAgent with a profile the daemon doesn't recognize | mitigate | Daemon's launch_agent validates the profile against policy.json; returns Err on unknown profile → agent_cli propagates the error |
| T-74-05-02 | Denial of Service | nono agent launch blocks indefinitely waiting for daemon pipe | mitigate | Set a connection timeout on the control pipe CreateFileW call (e.g. 5s); return Err if daemon not responding |
| T-74-05-03 | Information Disclosure | `nono agent list` exposes tenant SIDs to the terminal | accept | SIDs are per-user (same security context as the operator); this is the intended output; no cross-user leak |
| T-74-05-04 | Elevation of Privilege | `nono daemon install` grants incorrect service account | mitigate | `sc create` with `type= userservice` registers as per-user service (not machine-wide); acceptance criterion: "type= userservice" in the sc command |
| T-74-SC | Tampering | cargo installs | accept | No new packages |
</threat_model>
