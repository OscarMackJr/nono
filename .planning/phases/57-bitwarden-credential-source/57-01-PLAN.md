## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator env → bw/bws subprocess | BW_SESSION / BWS_ACCESS_TOKEN travel from parent-process env to the child process environment. Token is not in argv; it is accessible via /proc/<pid>/environ on Linux — acknowledged known limitation, same class as op://. |
| bw/bws CLI stdout → keystore.rs | Subprocess output is parsed as JSON and field values extracted. Untrusted in the sense that a corrupted CLI could return adversarial JSON. Mitigated by strict serde_json parse; any non-UTF-8 or non-JSON output is a hard error. |
| Operator-supplied URI → validate_bw_uri | URI is operator-supplied (from profile or --credential flag). Attacker who can control the URI string may attempt injection. Mitigated by FORBIDDEN_URI_CHARS + is_valid_bw_id allowlist before any spawn. |
| bw/bws stdout (Vec<u8>) → memory | The raw subprocess output Vec<u8> is not zeroized. This is a known, documented limitation acknowledged in code comments and in RESEARCH.md Pitfall 4. The same limitation exists for op://. After extraction, values are wrapped in Zeroizing<String>. |
