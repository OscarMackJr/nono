## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → git tag push | The tag push is the irreversible release trigger; the tagged commit's content is what ships |
| CI signing → public MSIs | Authenticode signing of wrapper AND embedded payloads via repo secrets |
| published release → downloaders | What the public installs; unsigned payloads would be a tampering hazard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-09 | Spoofing | CI signing material (D-02) | mitigate | Operator confirms WINDOWS_SIGNING_CERT[_PASSWORD] present before tagging; release.yml fail-closed guard blocks otherwise; NEVER fall back to the self-signed POC cert for a public release |
| T-61-10 | Tampering | MSI embedded payloads | mitigate | release.yml signs binaries pre-package + the Verify MSI payload signatures admin-extract gate (fail-closed); operator post-release spot-check re-confirms wrapper + payload are Valid (catches the v0.57.4-class defect) |
| T-61-SC | Tampering | release pipeline supply chain | mitigate | No package installs this phase (Package Legitimacy Audit N/A); the signing + payload-verify gates are the supply-chain controls; gate release success on the signed-artifact steps, not the cosmetic crates.io/homebrew jobs |
| T-61-11 | Repudiation | wrong-version / wrong-commit tag | mitigate | Tag only after 61-PRETAG-READINESS.md is green and `cargo build --workspace` shows 0.58.0; v2.9 annotation tag placed on the exact v0.58.0 commit; bash git tag ops run from the verified `main` repo root (worktree-CWD guard, see <verify_runtime_note>) |
