<objective>
Extend the `nono-shell-broker` binary to accept a `--no-pty` flag and bind the three passed pipe handles as the child's stdio via `STARTF_USESTDHANDLES`.

Purpose: The broker binary runs in the child context and is responsible for calling `CreateProcessAsUserW` with the Low-IL token. In no-PTY mode, the broker must propagate the three anonymous-pipe handles (stdin/stdout/stderr) to the child's stdio slots. Without `STARTF_USESTDHANDLES`, the child writes to the broker's console (or /dev/null) and the supervisor relay never receives output (Pitfall 7 in RESEARCH.md).

Output: `--no-pty` flag parsed into `BrokerArgs.no_pty`; run() sets `STARTF_USESTDHANDLES` with the three passed handles when `no_pty=true`; two new unit tests cover parse_args_no_pty_flag_accepted and parse_args_no_pty_absent_defaults_false; existing PTY/console path is structurally unchanged.
</objective>

| T-51B-02 | Elevation of Privilege | Low-IL child write-deny weakened by --no-pty mode | accept | STARTF_USESTDHANDLES only changes which fd the child writes stdout to; mandatory-label NO_WRITE_UP enforcement is at token/kernel level, entirely independent of stdio handle binding; the Low-IL token is still constructed by the broker via create_low_integrity_primary_token regardless of --no-pty |
