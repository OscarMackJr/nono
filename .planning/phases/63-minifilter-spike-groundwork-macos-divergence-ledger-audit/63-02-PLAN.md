<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dev host → disposable Azure VM | TESTSIGNING is enabled ONLY on the throwaway VM, never on the Win11 26200 dev host (V14 fail-secure) |
| test-signed driver → host kernel | The VM's `SERVICE_DEMAND_START` (D-06) means a bad driver won't auto-load at boot — BSOD + snapshot = instant rollback, not a brick |
| nono maintainer → Microsoft (altitude authority) | The altitude request crosses to an external authority; the band choice avoids EDR collision |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-06 | Denial of Service | TESTSIGNING/HVCI/Secure-Boot triad (silent load failure) | mitigate | Standard-security-type VM (Secure Boot OFF, HVCI off) + the SC1 `msinfo32`/`bcdedit /enum all` capture documents the state so Phase 64 doesn't hit a silent-load wall (Pitfall 4) |
| T-63-07 | Tampering / Elevation | altitude band selection | mitigate | Request the FSFilter Activity Monitor band (360000-389999) from Microsoft; the INF/DESIGN.md record the AV-range (320000-329998) avoidance — prevents EDR collision (Pitfall 5) |
| T-63-08 | Elevation | TESTSIGNING enabled on a host | mitigate | TESTSIGNING is VM-only and the VM is disposable/snapshot-able (D-04); the dev host is never test-signing-enabled (V14) |
| T-63-09 | Tampering | spike `.sys` leaking into the repo / MSI | mitigate | The `.sys` is a VM-local throwaway, never committed (Pitfall 6); only the SC1 evidence text is committed; the `nono-wfp-driver.sys` placeholder + MSI are untouched |
