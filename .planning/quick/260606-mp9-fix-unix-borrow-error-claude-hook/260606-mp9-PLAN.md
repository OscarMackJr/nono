# Quick Task 260606-mp9 — Fix Unix-only E0716 borrow error in `claude_code_hook.rs`

## Problem

The `v0.62.0` (v2.9) `release.yml` run (GitHub Actions run `27031289871`, 2026-06-05)
**failed and published no release**. Root cause: all 4 non-Windows build-matrix jobs
(aarch64-linux, x86_64-linux, aarch64-darwin, x86_64-darwin) failed to compile with a
single error, gating the `Create Release` job (`needs:` all builds):

```
error[E0716]: temporary value dropped while borrowed
  --> crates/nono-cli/src/claude_code_hook.rs:409:38
```

The offending code is in `wrapped_bash_command`, which is `#[cfg(not(target_os = "windows"))]`
— so it never compiles on the Windows dev host, which is why the bug went undetected since
it was introduced in commit `7488dbba` (PR #4, Phase 60, 2026-05-29). The Windows x86_64
build compiled + signed cleanly; only the Unix/macOS targets are affected.

Current (broken) line 409:
```rust
let nono_exe = shlex::try_quote(&nono_exe.display().to_string())
    .map_err(|e| NonoError::HookInstall(format!("failed to quote nono executable: {e}")))?;
```

`nono_exe.display().to_string()` is a temporary `String`; `shlex::try_quote` returns a
`Cow<str>` borrowing it; the temporary drops at end-of-statement while the `Cow` is still
held → E0716.

## Task 1 — Bind the temporary so it outlives the `Cow` borrow

**File:** `crates/nono-cli/src/claude_code_hook.rs` (function `wrapped_bash_command`, ~line 409)

**Action:** Introduce a named binding for the `String` before quoting, so it lives to the
end of the function:

```rust
let nono_exe_display = nono_exe.display().to_string();
let nono_exe = shlex::try_quote(&nono_exe_display)
    .map_err(|e| NonoError::HookInstall(format!("failed to quote nono executable: {e}")))?;
```

Do **not** change any other behavior. The `command` quote on the following line already
borrows from a `&str` argument that outlives the statement and is fine — leave it as-is
unless clippy flags it.

**Verify:**
- `cargo clippy -p nono-cli --target x86_64-unknown-linux-gnu -- -D warnings -D clippy::unwrap_used` → clean (this is the target that failed in CI; both cross-targets are installed on this host).
- `cargo clippy -p nono-cli --target x86_64-apple-darwin -- -D warnings -D clippy::unwrap_used` → clean.
- `cargo clippy -p nono-cli -- -D warnings -D clippy::unwrap_used` (default Windows target) → still clean (regression guard; this path does not compile the cfg'd function but must not regress).

If a cross-target clippy cannot run (toolchain/linker missing for a target), record it as
PARTIAL + CI-deferred per CLAUDE.md §Coding Standards and note which target was deferred.
The **x86_64-unknown-linux-gnu** check is the decisive one — it is the exact target whose
build failed in run `27031289871`.

**Done when:** The Linux cross-target clippy compiles `claude_code_hook.rs` clean (E0716
gone), and no new warnings are introduced on any attempted target.
