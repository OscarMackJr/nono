## Overview

The composition milestone that makes nono's confinement **engine-neutral** — any AI agent engine
(not just Claude Code) is confined through one launch path — and adds a persistent multi-tenant
daemon, an unforgeable AI_AGENT identity, and supplementary controls. Built over three existing
subsystems (broker-arm launch, the SDDL capability pipe, the `nono-wfp-service` shape) plus one
net-new marker. User-mode only (ADR-65 No-go on kernel driver). **12/12 requirements satisfied;
milestone audit PASSED 2026-06-16.**

## Phases

### Phase 71: Engine-Agnostic Launch Productionization
**Goal**: Parent-and-confine any covered AI agent engine through one engine-neutral launch path —
the spike-003 path promoted to a first-class, de-spiked code path that every later phase consumes.
**Depends on**: Nothing (foundation) · **Requirements**: ENG-01, ENG-02, ENG-03
**Outcome**: VERIFICATION passed + live Win11 Aider end-to-end UAT. `windows_interpreters` profile
field; fail-secure exe/interpreter coverage gate (`validate_launch_paths`); R-B3 `path_has_write_owner`
pre-launch gate; `--workspace` absolute-grant + child-CWD; nested-job hardening (P6).
Plans: 71-01 (interpreters field + aider/langchain profiles), 71-02 (GLE-5 foreign-job diagnostic),
71-03 (coverage + write-owner primitives), 71-04 (CLI integration), 71-05 (HUMAN-UAT + Aider gate).

### Phase 72: nono-py Binding + In-Process-Exec Proof
**Goal**: Prove the abstraction in code — a Python/LangChain agent confined through `nono-py` with no

**Depends on**: Phase 71 · **Requirements**: ABI-01, ABI-02
**Outcome**: VERIFICATION passed + Win11 pytest UAT. `confined_run` (Shape A) + `confine` (Shape B,
born-confined broker re-exec); nono pin 0.57→0.62; `proj/DESIGN-engine-abstraction.md` E1-E5 contract
(+ zt-infra E5 mapping). Binding lives in sibling repo `../nono-py` (branch `44-broker-ffi-lockstep`).
Plans: 72-01 (Shape-B soundness spike + SC reword), 72-02 (binding impl), 72-03 (E1-E5 contract), 72-04 (example + UAT).

### Phase 73: AI_AGENT Marker
**Goal**: Every confined agent carries an unforgeable `AI_AGENT` identity bound to its spawn-time
token SID — non-agent can't claim it, agent can't shed it.
**Depends on**: Phase 71 · **Requirements**: MARK-01
**Outcome**: VERIFICATION passed 2026-06-16 (verification gate closed during the milestone audit —
code-complete since 2026-06-14, SC4 host-gated tests run 2026-06-16). `nono::AgentRegistry` +
`AgentClassification` anchored to the per-run AppContainer package SID; `nono classify <PID>` verb
(structural/non-authoritative cross-process by design — authoritative path is Phase 74+ daemon);
job-ACL hardening (`JOB_OBJECT_LIMIT_BREAKAWAY_OK` never set, SD denies Low-IL). Authoritative proof:
`sc4_classify_real_agent` (real child → AiAgent) + `sc4_classify_spoof_not_agent` (spoof → NotAnAgent).
Plans: 73-01 (AgentRegistry/classification), 73-02 (job SDDL hardening), 73-03 (classify verb + mint→registry + SC4 tests + SC5 docs).

### Phase 74: Persistent Multi-Tenant Daemon  *(marquee / riskiest — former spike 004)*
**Goal**: A least-privilege persistent daemon launches/confines multiple concurrent agents over one
tenant-isolated control pipe, fresh token+job per agent, deterministically reaped, split from the
elevated WFP service.
**Depends on**: Phase 71 (working single-launch — hard gate) + Phase 73 (marker) · **Requirements**: DMON-01/02/03
**Outcome**: human UAT PASS (SC1-SC5) on real Win11 26200. `nono-agentd` 2nd `[[bin]]` USER-service;
`control_loop.rs` serves `\\.\pipe\nono-agentd-control` (Low-IL-denying SDDL); per-tenant SID auth via
`ImpersonateNamedPipeClient`; fresh AppContainer+job per agent; deterministic reap; privilege-model ADR.
The terminal UAT caught the marquee gap (control plane unwired) → gap-closure 74-07/74-08.
Plans: 74-01 (ADR + spike harness), 74-02 (lib primitives), 74-03 (daemon skeleton), 74-04 (accept loop + launch),
74-05 (CLI verbs), 74-06 (HUMAN-UAT), 74-07 (control-pipe server), 74-08 (clean-detach + bare-exe + list polish).
