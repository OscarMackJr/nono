# Requirements: nono — v2.8 UPST7 + v2.7 Drain & Release

**Archived:** 2026-06-06 — milestone shipped
**Defined:** 2026-05-28
**Core Value:** Windows security must be as structurally impossible and feature-complete as Unix platforms; every nono command that works on Linux/macOS should work on Windows with equivalent security guarantees, or be explicitly documented as intentionally unsupported with a clear rationale.

## v2.8 Requirements (10 total)

Re-released as **v0.57.5** (tag `a3927be0`); CI confirmed every MSI-embedded executable verifies `MSI payload Authenticode OK`, and the operator confirmed on a live install. See `.planning/phases/53-release-drain/53-04-SUMMARY.md`.

### Network Filtering (NET)

- [x] **REQ-NET-01**: `--allow-domain` accepts a URL with **path scoping** and **fine-grained HTTP method + path restrictions**, enforced in `nono-proxy` (`route`/`filter`/`server`), and TLS-intercept endpoint rules are evaluated **before** credential selection. Cross-platform; the change is diff-inspected against the fork-divergent TLS-interception surface (Phase 34 C11 `fork-preserve`) rather than blind-cherry-picked. `nono why --host` awareness of the new scoping is preserved.

### Credentials (CRED)

- [x] **REQ-CRED-01**: A `bw://` Bitwarden credential source resolves secrets through the keystore abstraction alongside the existing `keyring://` / `env://` / `file://` schemes, with secret fields held in `Zeroizing<String>` (aligned with the fork's `zeroize` standard) and in-place truncation. Cross-platform; isolated surface in `crates/nono/src/keystore.rs`.

### Session Hooks (HOOK)

- [x] **REQ-HOOK-01**: A `session_hooks` profile field runs vetted hooks at session start/stop. On Unix the upstream `hook_runtime` behavior is preserved (gated unix-only as upstream ships it); on Windows the hooks execute via a Windows-safe design (broker-spawned, Low-IL, no `fork`/`sh` assumption) documented in an ADR. Hook resolution or execution failure is **fail-closed**, never silently skipped.

### Supervisor IPC (IPC)

- [x] **REQ-IPC-01**: The supervisor survives a transient child IPC close (keep-alive instead of dropping the supervisor loop), enforces bounded read-timeouts, and accepts connections robustly. The Unix side absorbs upstream's named-socket hardening (cross-platform-core); the Windows side translates the robustness intent onto the fork's Named-Pipe AIPC path (Phase 18) — a translate-not-cherry-pick.
