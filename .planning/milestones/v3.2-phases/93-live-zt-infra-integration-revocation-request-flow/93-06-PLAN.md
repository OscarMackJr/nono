<reachability_note>
Plan 01 makes the nono-py verify path **policy-authoritative**: a token whose `key_id` is absent from HKLM `Override\KmsPublicKeys\<key_id>` returns `Err(KeyNotAllowlisted)` (fail-closed, D-05) BEFORE any live check runs. The Phase 91 test keypair is "never a production trust root" (93-CONTEXT line 68). Therefore, without an explicit trust-root seed, the gate's allow branch is UNREACHABLE — every mint would `KeyNotAllowlisted` before reaching the provisioner. This plan's Task 1 seeds the test pubkey + signer ARN into the policy store via a **registry test-seam** (HKCU first, falling back to HKLM if the gate runs elevated) — NOT an env override, because D-05 forbids env from widening trust roots. The seed is torn down in a `finally` block so the policy store is never left polluted. This makes the allow path exercisable when the provisioner is present; absent provisioner still SKIPs cleanly.
</reachability_note>
