<objective>
Disprove the #1132 / D-10 allow_domain-shadows-credential-catch-all scenario on the fork's
exact-prefix `RouteStore` model with a fork test (no code change expected).

Purpose: Upstream's #1132 bug is intrinsic to its `select_route`/`RouteSelection` host-ordering
abstraction, where an allow_domain endpoint route matching the same upstream HOST as a credential
catch-all could win and shadow it. The fork has no host-keyed selection and no catch-all: it
dispatches by EXACT path-prefix key, and allow_domain endpoint routes get a DISJOINT key namespace
(`_ep_{domain}`) vs credential routes (the service name). The shadow class cannot arise. Per the
phase's default-to-no-op method (D-01/D-10), prove this with a disproof test, record equivalence in
the 89-04 ledger addendum, and SKIP the `RouteSelection` refactor. Fix the fork's route resolution
DIRECTLY only if — against expectation — the test reproduces the shadow; never import upstream's
`RouteSelection` enum (D-10 LOCKS it out).

Output: One new `#[test]` in route.rs (D-10 disproof).
</objective>
