<objective>
Land the ONE real code fix of Phase 89: the #1197 / D-07 customCredentials activation-gate
fix in `crates/nono-cli/src/proxy_runtime.rs`, plus its regression test, plus the D-01
external_proxy-mapping equivalence test.

Purpose: The activation gate in `prepare_proxy_launch_options` checks `credentials`,
`network_profile`, `allow_domain`, and `upstream_proxy` but NOT `custom_credentials`, so the
proxy does not start when only `customCredentials` is configured — leaving credentials that
should be injected via the proxy unprotected. This is the only behavioral gap in Cluster F
(everything else is verify-equivalence). Fixing it is fail-secure: it only ADDS proxy
activation, never relaxes a denial.

Output: Edited activation predicate (both branches) + 3 new tests in proxy_runtime.rs.
</objective>
