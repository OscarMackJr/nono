      # ADR-86: Library-Boundary Convergence — Audit Stack and Structured Diagnostics to Core Crate
      **Status:** Accepted
      **Phase:** 86 — Library-Boundary Convergence
      **Date:** 2026-06-19
      **Authors:** Phase 86 planning

    ## Context section must establish:
    - Prior state: the library was a "pure sandbox primitive" — all audit logic and diagnostic UX lived in nono-cli.
    - Upstream v0.63.0..v0.64.0 (8 commits, Clusters A + B) moved the audit/attestation/ledger stack and
      the structured-diagnostics model into the core nono crate. The DIVERGENCE-LEDGER (Phase 85) dispositioned
      both clusters as will-sync / adopt-upstream (Phase 85 D-03, locked).
    - The fork faces the highest merge risk in its history (~2200 LOC relocated, FFI surface added, Windows
      diagnostic paths must be preserved). This ADR records the deliberate decision to converge rather than fork-preserve.

    ## Decisions section must record:
    Decision 1 — Adopt upstream's audit relocation (Theme A):
      - AuditRecorder, ledger, merkle/inclusion-proof, attestation sign/verify move into crates/nono/src/audit.rs.
      - nono-cli retains only thin wrappers (audit_integrity.rs, audit_ledger.rs, audit_attestation.rs,
        audit_session.rs redirect to nono::audit::*).
      - Unit tests move with the code per D-04 (tests live with their unit-under-test).
      - Rationale: maximum convergence → minimum reconciliation surface for the next upstream sync.

    Decision 2 — Adopt upstream's structured-diagnostics relocation (Theme B):
      - DiagnosticFormatter (UX) moves OUT of core into crates/nono-cli/src/diagnostic/formatter.rs (4ad8ba92).
      - Structured diagnostic facts/codes (NonoDiagnostic, SessionDiagnosticReport, etc.) arrive as a
        6-file module directory crates/nono/src/diagnostic/ (a6aa5995).
      - NonoError gains diagnostic_code() and remediation() methods.
      - 3 new pub extern "C" fns expose the diagnostic surface to FFI clients.
      - Rationale: library becomes leaner (no UX code), FFI clients gain structured diagnostic access.

    Decision 3 — Windows denial paths: preserve-and-bridge, NOT converge (D-02):
      - exec_strategy_windows/{launch,network,mod}.rs are UNCHANGED (zero LOC rewritten).
      - Windows denial paths gain diagnostic_code()/remediation() "for free" as methods on existing NonoError
        variants — no new code required.
      - Full routing of Windows denial output end-to-end through the new core structured-diagnostics model
        is deferred (high regression risk, out of Phase 86 scope).
      - This is the SINGLE deliberate fork carve-out from upstream's boundary line.

    Decision 4 — Verification gate must be --workspace --all-targets (D-05):
      - The nono-ffi crate (cdylib/staticlib) is NOT reachable via --bin nono scope.
      - NonoDiagnosticCode repr-C enum adds exhaustive-match requirements; --bin nono hides E0004.
      - Gate must be: cargo clippy --workspace --all-targets -- -D warnings -D clippy::unwrap_used.

    ## Consequences section must record:
    - The library is NO LONGER a pure policy-free sandbox primitive with no audit/diagnostic logic.
    - New invariant: library owns audit integrity/ledger/attestation/merkle AND structured diagnostic
      codes/facts (not UX, not rendering). CLI owns all rendering, flag suggestions, interactive UX.
    - FFI clients (nono-py, nono-ts) gain structured diagnostic access via 3 new extern-C fns.
    - Future upstream syncs: the library/CLI boundary is now aligned with upstream's boundary, reducing
      per-sync reconciliation friction.

    ## Alternatives Considered (table):
    | Alternative | Reason rejected |
    |---|---|
    | Fork-preserve: keep audit + diagnostics in nono-cli | Creates growing divergence from upstream; every future upstream sync accumulates more conflict debt on these surfaces. Rejected per Phase 85 D-03 (will-sync locked). |
    | Split: move only audit OR only diagnostics | Half-convergence still leaves heavy conflict debt; the two themes are architecturally coupled (diagnostics use audit records). Rejected for the same friction reason. |
    | Full Windows convergence (route Windows denial through new structured model) | High regression risk (SC#3 requires no regression in Windows denial output); the bridge pattern is sufficient for Phase 86. Deferred — see Decision 3. |

    Future sync guidance (as a callout paragraph in Consequences or as its own ## section):
    "The Windows denial path (exec_strategy_windows/) is a deliberate fork carve-out — it is NOT
    drift. Future upstream sync audits must recognize D-02 as intentional. The next upstream sync
    should re-evaluate whether full Windows convergence (routing denial output through the new core
    structured-diagnostics model) is feasible and desirable."

    Sign the commit: `git add proj/ADR-86-library-boundary-convergence.md && git commit -s -m "docs(86): add ADR-86 library-boundary-convergence ADR"`
