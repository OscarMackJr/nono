## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → core library | CLI thin wrappers call nono::audit::* after relocation; no privilege boundary change, but audit integrity must be preserved in transit |
| Core crate audit functions → merkle hash | Audit hash functions are security-critical; the sha2 crate performs the cryptographic work — any collision or truncation would undermine audit log integrity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86A-01 | Tampering | merkle/inclusion-proof in nono::audit | mitigate | Tests in #[cfg(test)] within audit.rs verify correctness of inclusion proofs; D-04 moves them with the code — verify test presence post cherry-pick |
| T-86A-02 | Tampering | ledger append+verify in nono::audit | mitigate | Ledger tests must pass in core crate after relocation; thin-wrapper shape prevents CLI from bypassing core ledger logic |
| T-86A-03 | Repudiation | attestation sign/verify moved to core | mitigate | Attestation unit tests (sign/verify) move to audit.rs per D-04; acceptance criterion requires cargo test -p nono attest passes |
| T-86A-04 | Tampering | cherry-pick conflict resolution | mitigate | Resolution target is upstream's end-state (86-RESEARCH.md answer-key pattern); executor uses `git show <SHA>` as authoritative reference for every contested hunk |
| T-86A-SC | Tampering | git cherry-pick provenance | mitigate | -x flag adds upstream SHA tracer; `git commit --amend -s` appends fork DCO; both verified in acceptance criteria |
