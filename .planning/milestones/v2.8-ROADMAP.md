## Overview

v2.8 is a **drain-then-sync** milestone mirroring the v2.5/v2.6 shape. The first three requirements (REQ-RLS-01/02, REQ-DRN-01/02) shipped the post-v2.7 untagged fixes as a properly signed release: Phase 53 uncovered and fixed a `release.yml` signing-order defect (MSI wrapper was signed but embedded payload binaries were not), re-released as **v0.57.5** (tag `a3927be0`), and closed the WFP service-stop/uninstall live-verify UAT. Phases 54–55 audited and absorbed UPST7 (upstream `v0.57.0..v0.59.0`, 40 commits / 14 clusters). Phase 56 added fine-grained `allow_domain` path+method filtering in `nono-proxy` with TLS-intercept endpoint-rules-before-credential-selection ordering. Phase 57 added `bw://` Bitwarden credential source with `Zeroizing<String>` posture. Phase 58 shipped `session_hooks` with Unix upstream behavior preserved and a Windows ADR-backed broker-spawned Low-IL design. Phase 59 hardened supervisor IPC with keep-alive, bounded read-timeouts, and robust accept on both Unix (cherry-picked) and Windows (translated).

**Audit verdict (v2.8-MILESTONE-AUDIT.md):** `tech_debt` — 10/10 requirements satisfied, 0 integration blockers, `cargo build --workspace` clean, two passing operator UATs (Phases 58 and 59) on real Win11 hardware (build 26200). Accumulated fail-secure advisory warnings and one tracked Windows defense-in-depth gap (Phase 58 D-05: Medium-IL hooks, ADR-tracked) remain for a follow-up pass.
